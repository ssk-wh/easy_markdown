# 构建说明

本文面向**开发者**，说明如何从源码构建 SimpleMarkdown，以及打包和诊断常见问题。

> 产品级规范见 `specs/00-产品愿景.md`，架构说明见 `specs/10-系统架构.md`。

---

## 前置依赖

### Windows
- Visual Studio 2019 或更高版本（MSVC 编译器）
- Qt 5.15.x（`msvc2019_64` 组件）
- CMake ≥ 3.16
- Git（用于 submodule）
- NSIS（用于打包）

### Linux
- GCC 9+ 或 Clang 10+
- Qt 5.15 开发包（`sudo apt install qtbase5-dev`）
- CMake ≥ 3.16
- Git
- `dpkg-dev` 与 `debhelper`（用于生成 deb 包）

---

## 获取源码

```bash
git clone --recursive https://github.com/ssk-wh/simple_markdown.git
cd simple_markdown
```

如果已 clone 但未初始化 submodule：

```bash
git submodule update --init --recursive
```

## 编译

### Windows 快捷脚本

```bat
build_on_win.bat release    :: Release 构建
build_on_win.bat clean      :: 清理后构建
```

### Linux 快捷脚本

```bash
./build_on_linux.sh release
```

### 原始 CMake 命令

```bash
# Windows (MSVC)
cmake -S . -B build
cmake --build build --config Release

# Windows (Ninja)
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build

# Linux
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j$(nproc)
```

## 打包

> **一键打包**：`pack_on_win.bat` / `pack_on_linux.sh` 会自动先调用对应的 `build_on_*` 脚本（release 模式）完成编译，再收集依赖、生成安装包。不需要再手动先跑 `build_on_*`。
>
> - 只想编译 → 运行 `build_on_*`
> - 一键打包（含编译）→ 运行 `pack_on_*`
>
> 若 pack 内部的编译阶段失败，脚本会立即以非零退出码终止，**不会**继续执行依赖收集与打包步骤。完整规范见 `specs/30-构建与发布流程.md`。

### Windows（NSIS 安装包）

```bat
pack_on_win.bat
```

脚本内部执行：
1. 调用 `build_on_win.bat release` 进行编译（失败立即终止）
2. 调用 `installer\collect_dist.py` 收集依赖（Qt DLL、插件、翻译、资源）到 `installer\dist\`
3. 调用 NSIS：`makensis installer\SimpleMarkdown.nsi`
4. 生成 `installer\SimpleMarkdown-<version>-Setup.exe`

### Linux（deb 包）

```bash
./pack_on_linux.sh
```

脚本内部执行：
1. 调用 `./build_on_linux.sh release` 进行编译（失败立即终止）
2. 从 `CHANGELOG.md` 提取版本号生成 `debian/changelog`
3. 调用 `dpkg-buildpackage`

### 版本号管理

- 版本号源头：`CHANGELOG.md` 的最新 `## [x.y.z] - YYYY-MM-DD` 条目
- NSIS 打包和 deb 打包都从 CHANGELOG 提取，保持 Windows/Linux 一致

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| CMake cache 过期 | 删除 `build/CMakeCache.txt` 后重建 |
| 编译找不到 Qt | 检查 `CMakeCache.txt` 中的 `Qt5_DIR` 路径 |
| 安装包大小异常 | 运行 `installer/collect_dist.py` 检查是否包含了 debug DLL |
| 安装包无法启动 | 检查 exe 时间戳（见下方诊断流程） |

## 打包产物过期问题诊断

**关键教训**：打包时使用了旧的编译产物（exe 时间戳为过去的日期），会导致安装包包含过时代码，应用无法启动。

### 编译 → 打包的完整流程

> 自 2026-04-13 起，`pack_on_win.bat` / `pack_on_linux.sh` **已经自动包含步骤 2**（内部先调用 `build_on_* release`）。
> 下面的分步视图保留用于诊断：当你怀疑产物过期时，可以单独跑每一步确认时间戳。

```
1. 代码修改
   ↓
2. 编译：build_on_win.bat release      (pack_on_win.bat 内部会自动执行这一步)
   ✓ 检查：build\src\app\SimpleMarkdown.exe 时间戳 = 当前日期时间
   ✓ 如果时间戳是旧日期，说明之前编译失败，需要重新编译
   ↓
3. 收集依赖：python installer\collect_dist.py
   ✓ 自动 copy 最新的 exe 到 dist/
   ✓ 检查：installer\dist\SimpleMarkdown.exe 时间戳应与上面一致
   ↓
4. 打包：makensis installer\SimpleMarkdown.nsi
   ↓
5. 验证：运行 installer\dist\SimpleMarkdown.exe 测试启动
   ✓ 应该能正常启动应用界面
```

### 诊断"无法启动"问题

```bat
REM 检查 exe 时间戳
for /F %%A in ('powershell -Command "(Get-Item build\src\app\SimpleMarkdown.exe).LastWriteTime.ToString()"') do echo Build exe: %%A
for /F %%A in ('powershell -Command "(Get-Item installer\dist\SimpleMarkdown.exe).LastWriteTime.ToString()"') do echo Dist exe: %%A

REM 如果时间戳不一致或都很旧，需要重新编译：
build_on_win.bat clean
build_on_win.bat release
python installer\collect_dist.py

REM 然后测试
installer\dist\SimpleMarkdown.exe
```

## 验证修复时必须先关闭旧进程

本应用采用**单实例模式**：第二个进程会把命令行参数转发给已有进程后自身退出。这意味着开发期间反复编译验证时，如果不先关掉已运行的实例，新启动的 exe 实际上是把控制权交给旧版本，看到的不是新改动。

完整的 build → 验证流程：

1. 编译：`build_on_win.bat release`（或 `pack_on_*` 一键打包）
2. 收集依赖：`python installer\collect_dist.py`
3. **关闭已运行的实例**：
   ```cmd
   taskkill /F /IM SimpleMarkdown.exe 2>NUL
   ```
   或 PowerShell：
   ```powershell
   Get-Process SimpleMarkdown -ErrorAction SilentlyContinue | Stop-Process -Force
   ```
4. 启动新进程：`installer\dist\SimpleMarkdown.exe`
5. 验证

Linux 类似：`pkill -f SimpleMarkdown` 然后启动新进程。

### 打包前同样要关闭旧进程

`pack_on_win.bat` / `pack_on_linux.sh` 内部第一步是 `installer/collect_dist.py`，它会**清空 `installer/dist/`** 重新拷贝。如果 SimpleMarkdown.exe 还在运行，dist 下的 DLL（`MSVCP140.dll` / `Qt5Core.dll` 等）被进程占用，`shutil.rmtree` 会抛 `PermissionError: [WinError 5] 拒绝访问` 导致打包失败：

```
PermissionError: [WinError 5] 拒绝访问。: '...\dist\MSVCP140.dll'
[ERROR] Dependency collection failed
```

**所以打包前同样要 taskkill**：

```cmd
taskkill /F /IM SimpleMarkdown.exe 2>NUL
.\pack_on_win.bat
```

完整开发循环：

```
改代码 → 杀进程 → 编译 → collect / pack → 启动 → 验证 → （改下一轮前）杀进程
```

## 代码编辑注意事项

### 源码编码

所有 C++ 源文件统一使用 **UTF-8 无 BOM**（项目现状）。修改已有文件时严禁变更 BOM 状态。若遇到 GBK 旧文件，先转成 NO-BOM UTF-8 再编辑：

```python
with open(path, 'rb') as f:
    text = f.read().decode('gbk')
with open(path, 'wb') as f:
    f.write(text.encode('utf-8'))  # 不写 BOM
```

### Windows 控制台中文

可能输出中文的命令前加 `chcp.com 65001 > /dev/null 2>&1;`：

```bash
chcp.com 65001 > /dev/null 2>&1; build_on_win.bat release
```

纯英文命令不加。
