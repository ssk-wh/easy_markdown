# 全局约束与不变量

## 1. 目的

汇总所有**必须遵守的工程级硬约束**。这些约束由产品愿景（00）或系统架构（10）派生，违反即视为 bug。所有模块 Spec 在 `depends` 中引用本文。

N/A（约束集合）

## 3. 行为契约

### 技术栈锁定

- **[INV-LANG]** 代码必须是 **C++17**，不得使用 C++20/23 特性（与最低支持的 MSVC 2019 兼容）
- **[INV-QT]** GUI 框架必须是 **Qt 5.15 LTS**
- **[INV-CMAKE]** 构建系统必须是 **CMake ≥ 3.16**
- **[INV-PARSER]** Markdown 解析器必须是 **cmark-gfm**，不得更换
- **[INV-HILITE]** 代码高亮必须是 **KSyntaxHighlighting**
- **[INV-TEST]** 单元测试框架必须是 **GoogleTest**

### 跨平台

- **[INV-OS-PARITY]** Windows 和 Linux 的功能必须等价；不允许任一平台独有的关键功能
- **[INV-OS-PATH]** 路径处理必须使用 `QString` / `QDir`，禁止手写 `/` 或 `\` 分隔符
- **[INV-OS-LINE-ENDING]** 保存文件时保留原始换行符风格（CRLF 或 LF），新建文件使用平台默认

### 线程安全

- **[INV-THREAD-MAIN]** 所有 QWidget 方法、QPainter、GUI 事件处理必须在主线程
- **[INV-THREAD-PARSER]** Markdown 解析必须在后台线程
- **[INV-THREAD-IMMUTABLE]** 跨线程共享的对象必须是 immutable 或加锁保护
- **[INV-THREAD-QUEUED]** 跨线程 signal-slot 必须用 `Qt::QueuedConnection` 或 `Qt::AutoConnection`
- **[INV-THREAD-NO-WIDGET-IN-BG]** 后台线程禁止直接构造 QWidget/调用 QPainter/访问 QPaintDevice

### 高 DPI（详见 `横切关注点/40-高DPI适配.md`）

- **[INV-DPI-METRICS]** 所有 `QFontMetricsF` 必须传入 `QPaintDevice*`：
  - 布局阶段：`QFontMetricsF(font, device)`
  - 绘制阶段：`QFontMetricsF(font, p->device())`
  - 鼠标命中：`QFontMetricsF(font, viewport())`
- **[INV-DPI-CTOR]** 构造函数中禁止计算依赖 DPI 的度量值（此时无 device），必须延迟到 `updateMetrics(device)`
- **[INV-DPI-CONSISTENT]** 布局阶段和绘制阶段必须使用同一度量系统，不得混用逻辑像素和物理像素
- **[INV-DPI-NO-HARDCODE]** 禁止硬编码 padding/margin 像素值，必须用 `fm.height() * ratio` 表达

### 坐标系统

- **[INV-COORD-ABS]** 子块绝对坐标 = 父块绝对坐标 + 子块相对偏移，递归绘制时必须显式计算：
  ```cpp
  qreal childAbsX = absX + child.bounds.x();
  qreal childAbsY = absY + child.bounds.y();
  ```
- **[INV-COORD-NO-INHERIT]** 严禁把父块的 `absX/absY` 直接传给子块

### 性能

- **[INV-PERF-TYPING]** 打字延迟 < 3 ms
- **[INV-PERF-PREVIEW]** 预览更新 < 80 ms（含 30 ms 防抖）
- **[INV-PERF-STARTUP]** 冷启动 < 500 ms
- **[INV-PERF-MEMORY]** 空载内存 < 30 MB
- **[INV-PERF-NO-BLOCK]** 主线程单次事件处理 < 16 ms

### 安全与隐私

- **[INV-SEC-OFFLINE]** 禁止主动发起网络请求（除非明确的用户操作，如加载远程图片；MVP 不支持）
- **[INV-SEC-NO-TELEMETRY]** 禁止收集用户数据或发送遥测
- **[INV-SEC-NO-SECRETS]** 禁止硬编码任何密钥、Token、URL
- **[INV-SEC-EXT-FILES]** 打开外部文件时必须防御路径穿越和符号链接攻击

### 代码质量

- **[INV-CODE-UTF8]** 所有 C++ 源文件必须是 **UTF-8 无 BOM** 编码（项目现状，保持一致；修改已有文件时严禁变更 BOM 状态）
- **[INV-CODE-RAII]** 严禁裸 `new`/`delete`，必须用智能指针或 RAII
- **[INV-CODE-NO-EXCEPTIONS-CROSS-MODULE]** 模块边界不抛异常，用返回值/optional 表达失败
- **[INV-CODE-NO-SINGLETON]** 禁止新增全局单例（现有的主题、最近文件等保持现状）
- **[INV-CODE-CHANGELOG]** 任何影响可执行文件行为的改动（bug 修复、功能新增、性能优化、样式调整）必须在完成后立即更新 `CHANGELOG.md` 的最新条目；纯内部重构/Spec 梳理可不记，但重大架构转型必须记

### 外部依赖

- **[INV-DEP-NO-NET]** 构建期间不联网下载依赖（submodule 或 vendored）
- **[INV-DEP-VERSIONED]** 所有第三方库必须锁定版本
- **[INV-DEP-LICENSE]** 只接受 MIT / BSD / Apache 2.0 / LGPL 兼容的许可证

## 7. 验收条件

- [T-1] `grep -rn "new \|delete " src/` 无裸 new/delete（除第三方 headers）
- [T-2] `grep -rn "QFontMetricsF(.*)$" src/` 无缺失 device 参数的调用
- [T-3] `grep -rn "http" src/` 只在明确允许的位置（远程图片、文档链接）
- [T-4] 所有平台启动时间测试通过
- [T-5] UTF-8 编码检测脚本通过（源文件一律 NO-BOM UTF-8）
- [T-6] 近 10 个 commit 的改动在 CHANGELOG.md 中都有对应条目（手工抽查）

## 8. 已知陷阱

### 构造函数 vs updateMetrics 不一致
见 `横切关注点/40-高DPI适配.md` 第 5 节。

### QFontMetricsF 缺失 device 参数
这是本项目踩过最多次的坑。一次 grep 一次 fix，全局的唯一解法是：**所有 `QFontMetricsF` 必须显式带 device 参数**。

### Windows 路径反斜杠
不要用 `"\\"` 或 `"/"` 字面量拼路径，始终走 `QDir::separator()` 或 `QString::fromNativeSeparators`。

### 子块坐标错传
父 → 子递归绘制时，忘记加 `child.bounds.x() / y()` 是反复出现的 bug。所有容器块（List/BlockQuote/Table）的 paint 代码必须显式计算 `childAbsX/childAbsY`。
