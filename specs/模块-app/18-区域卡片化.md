# 主区域卡片化视觉（MVP · 已中止）

## 1. 目的

把编辑器 / 预览 / 目录三个主区域从"贴边直角"改为"轻度边框 + 圆角"，让三个区域在视觉上成为独立卡片，分层更清晰。

**V1 MVP 定位**：最小视觉改动，不改布局结构、不加阴影、不加新 Theme 字段。仅在 `applyTheme` 的 stylesheet 里追加 4 行 CSS 规则。

## 2. 输入 / 输出

- **输入**：当前主题的 `border`（= `editorGutterLine`）色值
- **输出**：
  - EditorWidget / PreviewWidget / TocPanel 三个 widget 外边框：1px 实线 + 6px 圆角
  - 边框色跟随主题切换
  - 不改 widget 布局 / 不改 splitter 结构 / 不改 Qt 子 widget 父子关系

## 3. 行为契约（不变量）

### [INV-1] 仅通过 stylesheet 实现

`applyTheme` 的深色分支与浅色分支**各追加一段** CSS：

```css
EditorWidget, PreviewWidget, TocPanel {
  border: 1px solid <border>;
  border-radius: 6px;
}
```

**禁止**：
- 改 `createTab` / `MainWindow` 的 splitter / layout 结构
- 给这些 widget 添加外层包装 QFrame
- 引入 `QGraphicsDropShadowEffect`（阴影效果可能在低端 Linux 下掉帧，MVP 不要）
- 引入新的 Theme 字段（MVP 复用 `editorGutterLine`）

### [INV-2] 类名选择器优先级覆盖 QAbstractScrollArea

`QAbstractScrollArea { border: none; }` 在 stylesheet 中先被设置；类名选择器 `EditorWidget` / `PreviewWidget` 优先级高于 `QAbstractScrollArea`，会覆盖 border-none 使边框显式渲染。

这依赖 Qt stylesheet 的 **类名选择器 > 基类选择器** 规则，不能调整两条规则的相对顺序。

### [INV-3] QAbstractScrollArea 的 viewport 不应破坏圆角可见性

EditorWidget / PreviewWidget 继承自 `QAbstractScrollArea`，它们各自的 `paintEvent` 使用 `painter.fillRect(viewport()->rect(), bgColor)` 填充背景。
viewport 区域在 frame 内部，frame 的 border-radius 通过 Qt 样式引擎在 frame 层绘制，
**允许**圆角四个角露出一点像素级的父控件背景色——视觉上表现为"卡片圆角 + 内部矩形内容"。
6px 圆角半径足够小，不会造成明显的内容被圆角裁切的视觉问题。

若未来加大到 16-20px 圆角（参考 iOS 26 Liquid Glass 视觉），需要配套给 viewport 做 clip 或给 viewport 也应用 radius——V2 再处理。

### [INV-4] 主题跟随

边框色必须从 Theme 字段派生（`m_currentTheme.editorGutterLine`），深色 / 浅色主题切换时边框色**立即** 刷新（通过 `applyTheme` 重新 setStyleSheet 触发）。

### [INV-STYLE-HOOK] 必须启用 stylesheet 渲染钩子（踩坑记录）

光在 stylesheet 中写 `EditorWidget { border: 1px solid ... }` **不够**，Qt 底层根据 widget 类型有不同的 border 绘制路径：

- **`QAbstractScrollArea` 子类（EditorWidget / PreviewWidget）**：默认 `frameShape = NoFrame`，stylesheet 的 border/border-radius 不会渲染。**必须** 在 `createTab` 中显式 `setFrameShape(QFrame::StyledPanel) + setFrameShadow(QFrame::Plain)` 让 Qt 把 frame 绘制委托给 stylesheet。
- **纯 `QWidget` 子类（TocPanel）**：默认不响应 stylesheet 的 background / border。**必须** 在其构造函数中 `setAttribute(Qt::WA_StyledBackground, true)`。

这两个 hook 缺一个都会让 border 在该 widget 上完全不可见。踩过坑（2026-04-15），记录备忘。

### [INV-5] 零布局影响

追加的 CSS 仅作用于 widget 自身的 border 属性，不影响：
- 三个区域的 `sizeHint` / `minimumSize` / 实际占据空间
- Splitter 分配的像素尺寸
- 滚动条行为
- 高 DPI 下的度量（border-radius 单位是逻辑像素，Qt 自动缩放）

如果某次改动后发现区域被挤压或 splitter sizes 错乱，说明本 INV 被违反，必须回退。

## 4. 实现细节

### 4.1 代码位置

`src/app/MainWindow.cpp::applyTheme(const Theme&)`：
- 深色分支（`if (theme.isDark)`）在"分割线"CSS 段之后、"弹窗"CSS 段之前插入
- 浅色分支（`else`）在"splitter"CSS 段之后、"滚动条"CSS 段之前插入

两段内容相同，共用相同的 border 色（`m_currentTheme.editorGutterLine.name()` 取自作用域内的 `border` 局部变量）。

### 4.2 选择器使用 class 名

Qt stylesheet 中 `EditorWidget` 作为类型选择器，匹配该 C++ 类的**所有实例**（不包括子类）。因为 EditorWidget / PreviewWidget / TocPanel 都是项目自定义类、没有子类，选择器精确到这三种 widget。

如果未来这些类被继承（例如 `class EnhancedEditorWidget : public EditorWidget`），需要用 `.EditorWidget, .PreviewWidget, .TocPanel` 带点号的子类也匹配，或把规则挪到基类 —— V1 MVP 不预见这种情况。

## 5. 性能预算

| 操作 | 目标 |
|------|------|
| 主题切换后重绘三个区域 | 沿用 `specs/横切关注点/30-主题系统.md` < 100ms，无额外开销 |
| border-radius 每帧绘制成本 | Qt 内部已缓存，可忽略 |

## 6. 验收条件

- **[T-CARD-1]** 启动应用，三个区域都可见到 1px 实线边框 + 轻微圆角
- **[T-CARD-2]** 切换主题（浅色 ↔ 深色）：边框色同步更新，圆角保留
- **[T-CARD-3]** 切换主题（内置 8 款 + 用户主题）：每款主题下边框色都来自对应 `editorGutterLine`，视觉和谐
- **[T-CARD-4]** 拖动分隔条调整布局：圆角不撕裂、边框跟随区域尺寸变化
- **[T-CARD-5]** 高 DPI（125% / 150% / 200%）下：border-radius 随缩放比自动调整，无"放大后圆角看起来像直角"现象
- **[T-CARD-6]** 回归 spec `12-主题插件系统.md INV-1`：切主题时所有 widget 同步、无残留旧色
- **[T-CARD-7]** 焦点模式（F11）：编辑器隐藏时预览仍保留圆角；退出焦点模式后编辑器圆角恢复

## 7. 非目标

- 不做卡片阴影（V2 可加，需评估低端 Linux 性能）
- 不做 18-20px 大圆角（需要 viewport clip 配合）
- 不做区域间 padding / 卡片外边距（会改布局）
- 不引入新 Theme 字段 `cardRadius` / `cardPadding` / `cardShadow`（V2）
