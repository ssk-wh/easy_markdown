# 状态栏布局（左编辑信息 / 右文件元数据 + 保存状态）

## 1. 目的

把状态栏从"所有信息混在一起堆右边"重构为**左 + 右两区**，符合用户从左到右"我在写什么 → 这个文件的状态"的阅读路径。

## 2. 输入 / 输出

- **输入**：
  - 当前 Tab 的 Document（filePath / isModified / detectedLineEnding / lastModified）
  - 当前主题对象（`m_currentTheme.name`）
  - 字符编码（项目硬约束 UTF-8）
  - 30s 周期 timer
- **输出**：
  - 状态栏 5 个左侧 label：光标位置 / 行数 / 字数 / 字符数 / 阅读时间
  - 状态栏 4 个右侧 label：编码 / 换行 / 主题名 / 保存状态
  - 自动保存失败时额外的瞬时红字提示（参见 14-自动保存 INV-2）

## 3. 行为契约（不变量）

### [INV-1] 左右分区

- 左区使用 `QStatusBar::addWidget(...)`：行号/列号 → 行数 → 字数 → 字符数 → 阅读时间
- 右区使用 `QStatusBar::addPermanentWidget(...)`：编码 → 换行 → 保存状态
- 中间空白由 QStatusBar 自动填充
- 顺序固定（不可在运行时调换），用户在不同主题/语言下都能稳定预期同一信息位于同一区位
- **历史 note（2026-04-15）**：右区最初还包含"主题名"label，因为切主题时整个 UI 颜色已明显变化、菜单中也有勾选指示，文字主题名属于冗余信息，**已移除**。

### [INV-2] 编码字段

- 当前固定显示 `UTF-8`
- 不依赖任何运行时检测——`INV-CODE-UTF8`（全局硬约束）保证所有源文件读写都是 UTF-8（见 `specs/20-约束与不变量.md`）
- 未来若支持 GBK/Latin-1 文件，需要更新 Document::detectedEncoding 后再展示

### [INV-3] 换行字段

- 来源：`Document::detectedLineEnding()`
  - `LF` 显示 `LF`
  - `CRLF` 显示 `CRLF`
- 切换 Tab、打开新文件后必须刷新

### [INV-4] ~~主题名字段~~（已移除）

~~来源：`m_currentTheme.name` / `applyTheme` 末尾 `updateRightStatusBar`~~
**2026-04-15 移除**：切换主题时整个 UI 颜色已明显变化，再加文字 label 多余。代码中 `m_statusThemeName` 成员已删。

### [INV-5] 保存状态字段（核心 UX）

`updateSaveStatusOnly()` 必须按以下三档优先返回：

1. **未命名文档** (`filePath().isEmpty()`):
   `tr("Unsaved (new file)")` → 中文 "未保存（新文件）" / 英文 "Unsaved (new file)"
   样式：默认色

2. **有路径但未保存修改** (`!filePath.isEmpty() && isModified()`):
   `tr("Unsaved changes")` → "未保存修改" / "Unsaved changes"
   样式：警示红 `#C62828`

3. **有路径且已保存** (`!filePath.isEmpty() && !isModified()`):
   显示相对时间："已保存 · X 分钟前" / "Saved · X min ago"
   - secs < 60：`Saved · just now` / "刚刚"
   - secs < 3600：`Saved · N min ago`
   - secs < 86400：`Saved · N hour ago`
   - secs >= 86400：`Saved · N day ago`
   样式：默认色
   时间戳来源：`QFileInfo(fp).lastModified()`

### [INV-6] 触发刷新的事件

`updateRightStatusBar()` / `updateSaveStatusOnly()` 必须在以下事件触发：

| 事件 | 调用函数 | 字段范围 |
|------|---------|---------|
| 切换 Tab | `updateRightStatusBar()` | 全部右区 |
| 主题切换 | `updateRightStatusBar()` | 全部（顺便刷主题名） |
| 文件 Save / Save As | `updateRightStatusBar()` (As 版本) / `updateSaveStatusOnly()` (Save 版本) | 保存状态 |
| 自动保存完成 | `updateSaveStatusOnly()` | 保存状态 |
| `Document::modifiedChanged` | `updateSaveStatusOnly()` | 保存状态 |
| 30s 周期 timer | `updateSaveStatusOnly()` | 保存状态（重算相对时间） |

### [INV-7] 30s 相对时间刷新

- `m_relTimeRefreshTimer` 在 `setupStatusBar()` 中启动，`interval=30000`，`singleShot=false`
- 每次 timeout 调 `updateSaveStatusOnly()`
- **不访问磁盘**——仅基于已读取的 `lastModified()` 做差值算
- timer 不能被销毁/暂停（除非应用退出）
- 即便用户关闭自动保存，该 timer 仍然继续——它只刷新文案，与自动保存无关

### [INV-8] 与自动保存 spec 的协同

- 自动保存失败时显示的红字 label `m_statusAutoSaveMsg` **属于右区**，位于保存状态 label 之后
- 自动保存成功时**不**显示任何额外提示——只通过 `m_statusSaveStatus` 文本变化（"未保存修改" → "已保存 · 刚刚"）让用户感知
- 这避免了"每次保存都弹个 toast"的噪声

### [INV-9] 主题跟随

- 状态栏背景由 `applyTheme` 中 `QStatusBar { background: ... }` 样式表设置
- 4 个右侧 label 的文字色继承 QStatusBar 默认（不设独立样式表，除"未保存修改"红字和"自动保存失败"红字两个例外）
- 切换主题后所有 label 颜色应自动更新

## 4. 接口

### 4.1 MainWindow 新增成员

```cpp
QLabel* m_statusEncoding   = nullptr;  // "UTF-8"
QLabel* m_statusLineEnding = nullptr;  // "LF" / "CRLF"
QLabel* m_statusThemeName  = nullptr;  // 当前主题 name
QLabel* m_statusSaveStatus = nullptr;  // 保存状态，按 INV-5 三档

void updateRightStatusBar();    // 全部右区一次性刷新
void updateSaveStatusOnly();    // 仅保存状态（高频）
```

### 4.2 setupStatusBar 改动

```cpp
auto* sb = statusBar();
auto makeLeftLabel = [sb](const QString& t) -> QLabel* {
    auto* l = new QLabel(t, sb);
    l->setContentsMargins(8, 0, 8, 0);
    sb->addWidget(l);   // 改为 addWidget
    return l;
};
auto makeRightLabel = [sb](const QString& t) -> QLabel* {
    auto* l = new QLabel(t, sb);
    l->setContentsMargins(8, 0, 8, 0);
    sb->addPermanentWidget(l);
    return l;
};

// 左区 5 个：保留原有顺序
m_statusCursorPos = makeLeftLabel(...);
...

// 右区 4 个：编码 → 换行 → 主题 → 保存状态
m_statusEncoding   = makeRightLabel("UTF-8");
m_statusLineEnding = makeRightLabel("LF");
m_statusThemeName  = makeRightLabel("");
m_statusSaveStatus = makeRightLabel(tr("Unsaved (new file)"));
```

## 6. 验收条件

- **[T-SB-1 左右分区]** 启动应用，状态栏左侧从左到右为 `Ln/Col` `Lines:` `Words:` `Chars:` `Read:`，右侧从右到左为 `保存状态` `主题名` `LF` `UTF-8`
- **[T-SB-2 新建未命名]** 新建 Tab，右下显示 `未保存（新文件）`（默认色）
- **[T-SB-3 打开已保存文件]** 打开一个未修改的 .md，右下显示 `已保存 · 刚刚`（默认色）
- **[T-SB-4 编辑变 dirty]** 在已保存文件中输入字符，右下立即变 `未保存修改`（红字 `#C62828`）
- **[T-SB-5 手动保存回到 saved]** 按 Ctrl+S，右下立即变回 `已保存 · 刚刚`
- **[T-SB-6 自动保存回到 saved]** 等待 1.5s 自动保存触发，右下变 `已保存 · 刚刚`
- **[T-SB-7 30s 相对时间刷新]** 保存后等待 60-90s，右下变 `已保存 · 1 分钟前`（不刷新磁盘，只重算文字）
- **[T-SB-8 切换 Tab 刷新右区]** 切换到另一个 Tab，主题名 / 换行 / 保存状态全部按新 Tab 的 Document 状态刷新
- ~~**[T-SB-9 切换主题刷新主题名]**~~ 已移除（主题名 label 被去掉）
- **[T-SB-10 CRLF 文件检测]** 打开一个 CRLF 编码的文件，状态栏右下显示 `CRLF`
- **[T-SB-11 i18n]** 切换 zh_CN / en_US，三种保存状态文本都正确翻译

## 7. 已知陷阱

### 7.1 lastModified() 的精度

`QFileInfo::lastModified()` 在 Windows / Linux / macOS 精度不同（NTFS 100ns，ext4 1ns，但 Qt 统一到秒），不影响相对时间计算。

### 7.2 与自动保存的失败提示位置冲突

`m_statusAutoSaveMsg`（红色失败提示）被 `addPermanentWidget` 加在保存状态 label **之后**，所以它出现时会"挤"到右边最右侧。这是预期行为——失败提示是临时的，5s 后会清空。

### 7.3 主题切换时的 label 颜色残留

如果用户在 dark 主题下编辑（"未保存修改"显示红字），切换到 light 主题后红字应该仍然可读。`#C62828` 在 light 和 dark 上都有足够对比度。**不要**针对主题切换重置 label 样式表——会让 INV-5 的 dirty 红字消失。

### 7.4 多 Tab 同步

切换 Tab 时只看**新当前 Tab** 的 Document，不查所有 Tab。其它 Tab 的 dirty 状态由它们各自的 tab title 标记体现，不在状态栏。

### 7.5 plurals

`Saved · %1 min ago` 等使用 Qt plural form (`numerusform`)。中文不区分单复数，英文 V1 也只用单数形式（"Saved · 5 min ago" 而非 "5 mins ago"），简化翻译维护。
