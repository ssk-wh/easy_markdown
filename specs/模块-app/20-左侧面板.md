# 左侧面板

## 1. 目的

管理左侧面板（`m_leftPaneSplitter`）的组成、显隐规则和宽度策略。左侧面板包含文件夹资源管理器和可选的侧边 Tab 栏，是用户浏览项目文件和切换标签页的核心区域。

## 2. 输入 / 输出

- 输入：用户打开/关闭文件夹、Ctrl+B 快捷键、Tab 栏位置菜单切换、演示模式切换
- 输出：左侧面板的可见性和宽度

## 3. 行为契约（不变量）

### 显隐规则

- [INV-LP-FOLDER-ONLY] 左侧面板**仅在有打开的文件夹时显示**，无论 Tab 栏位置设置如何。没有打开任何文件夹时，左侧面板始终隐藏
- [INV-LP-FOCUS-HIDE] 演示模式（F11）下左侧面板始终隐藏，优先级最高
- [INV-LP-CTRLB] Ctrl+B 手动隐藏侧边栏，状态为会话级（不持久化），重启后默认显示
- [INV-LP-AUTO-SHOW] 以下操作自动取消 Ctrl+B 的隐藏状态，恢复显示：
  - 打开新文件夹（菜单、拖拽、最近文件列表）
  - 切换 Tab 栏位置菜单项（Show All / Top Only / Side Only / Hide All）

### 判断优先级

```
演示模式？ → 隐藏（最高优先级）
Ctrl+B 手动隐藏？ → 隐藏
有打开的文件夹？ → 显示
否则 → 隐藏
```

### 宽度策略

- [INV-LP-WIDTH-RANGE] 左侧面板宽度范围：**窗口宽度的 1/8 ~ 1/6**
- [INV-LP-WIDTH-DEFAULT] 默认宽度为窗口宽度的 15%，夹紧到 [1/8, 1/6] 范围
- [INV-LP-WIDTH-RESTORE] 从隐藏变为可见时，若当前宽度低于窗口宽度 1/8，自动修正到默认值
- [INV-LP-WIDTH-USER] 用户拖拽 splitter 手动调整的宽度不受自动修正影响（仅在隐藏→可见转换时修正）

### Tab 栏位置

- [INV-LP-TAB-DEFAULT] 默认 Tab 栏位置为"全部显示"（侧边 + 顶部同时显示）
- [INV-LP-TAB-SIDE] 侧边 Tab 栏（`m_sideTabBar`）作为 `m_leftPaneSplitter` 的子 widget，位于文件夹面板下方
- [INV-LP-TAB-RATIO] 侧边模式下文件夹面板与 Tab 栏的默认高度比为 2:1

## 4. 接口（API 签名）

```cpp
// 切换侧边栏显隐（Ctrl+B）
void MainWindow::toggleSidebar();

// 统一的左侧面板显隐判断（所有影响左侧面板的操作最终都调用此函数）
void MainWindow::updateLeftPaneVisibility();

// Tab 栏位置切换
void MainWindow::setTabBarPosition(bool onSide, bool hideTopBar = true);
```

### 关键成员

```cpp
QSplitter* m_leftPaneSplitter;   // 左侧面板容器（竖向：文件夹 + 侧边 Tab）
QWidget* m_folderContainer;       // FolderPanel 的容器
FolderPanel* m_folderPanel;       // 文件夹资源管理器
SideTabBar* m_sideTabBar;         // 侧边 Tab 栏
QAction* m_toggleSidebarAct;      // Ctrl+B 菜单项
bool m_sidebarHidden;             // 用户手动隐藏标志（会话级）
bool m_tabBarOnSide;              // Tab 栏是否在侧边
```

## 5. 核心算法

### updateLeftPaneVisibility 逻辑

```
if 演示模式 → hide, return
if m_sidebarHidden → hide, return
if 有打开的文件夹 → show（并修正宽度）
else → hide
```

### 宽度修正（隐藏→可见转换时）

```
totalW = mainSplitter.width
minW = totalW / 8
maxW = totalW / 6
if sizes[0] < minW:
    sizes[0] = clamp(totalW * 15%, minW, maxW)
    sizes[1] = totalW - sizes[0] - sizes[2]  // 重分配给内容区
```

### showEvent 初始化（无保存状态时）

```
leftW = leftPane.isVisible ? clamp(totalW * 15%, totalW/8, totalW/6) : 0
tocW = clamp(totalW * 12%, 120, 240)
contentW = totalW - leftW - tocW
setSizes({leftW, contentW, tocW})
```

## 6. 性能预算

- 显隐切换：< 16ms（一帧内完成）
- 无额外内存分配（仅修改 QSplitter sizes）

## 7. 验收条件（测试用例清单）

- [T-1] 全新安装（无 QSettings）启动，左侧面板不显示（无文件夹打开）
- [T-2] 打开文件夹后，左侧面板出现，宽度在窗口宽度 1/8 ~ 1/6 之间
- [T-3] Ctrl+B 隐藏侧边栏，再按 Ctrl+B 恢复
- [T-4] Ctrl+B 隐藏后，打开新文件夹自动恢复显示
- [T-5] Ctrl+B 隐藏后，切换 Tab 栏位置菜单项自动恢复显示
- [T-6] 演示模式下左侧面板隐藏，退出演示模式后恢复
- [T-7] 关闭所有文件夹后左侧面板自动隐藏
- [T-8] 重启应用后 Ctrl+B 隐藏状态不保留（默认显示）
- [T-9] 用户拖拽调整宽度后，后续打开不会被自动修正覆盖

## 8. 已知陷阱

- **showEvent 时序**：`m_mainSplitter->restoreState()` 在 `showEvent` 中执行，晚于 `loadSettings`。恢复的 splitter 状态可能把左面板挤到 0 宽度，必须在 restoreState 后做 sanity check
- **setSizes 参数个数**：`m_mainSplitter` 有 3 个子 widget（左面板、内容、TOC），`setSizes` 必须传 3 个值，传 2 个会导致未定义行为
- **会话级 vs 持久化**：`m_sidebarHidden` 不写入 QSettings，避免用户"上次隐藏了侧边栏"后重启仍然隐藏造成困惑
