# 主题插件系统（Theme Plugin System）

## 1. 目的

在已有的亮/暗主题二选一基础上，提供一套**用户可自定义的主题插件系统**：

- 用户无需改代码、无需重新编译，只需编辑一个配置文件就能自定义整个 UI 风格
- 自定义范围**广**：颜色、字体、间距、边框、圆角、滚动条、分隔线、语法高亮、标记色 ……
- 主题以**文件**承载，便于在用户之间分享（复制粘贴 / 导出导入 / 社区共享）
- 内置主题（light / dark / system）继续存在，且也使用同一套 schema 表达，保证一致性
- 强制保证**安全性**：主题文件只允许承载数据，不允许执行代码

本 Spec 与 `specs/横切关注点/30-主题系统.md` 互补：
- 30-主题系统 定义"主题如何被应用到 UI"
- 本 Spec 定义"主题如何被外部定义、加载、验证"

## 2. 输入 / 输出

- **输入**：
  - 主题配置文件（格式待定，候选 TOML / YAML / JSON）
  - 文件来源：
    - 应用自带：`resources/themes/*.{toml|yaml|json}`
    - 用户目录：`%APPDATA%/SimpleMarkdown/themes/` (Windows) / `~/.config/SimpleMarkdown/themes/` (Linux)
    - 用户通过"导入主题文件"菜单选择的任意路径
  - 用户操作：选择主题、导入主题文件、导出当前主题、删除主题
- **输出**：
  - 运行时 `Theme` 实例（替代现有的 `Theme::light()` / `Theme::dark()` 硬编码）
  - 加载错误提示（状态栏或弹窗）
  - 主题列表（给菜单 / 设置页展示）

## 3. 行为契约（不变量）

### [INV-1] 配置文件格式

主题配置文件格式必须在**TOML / YAML / JSON**三者中择一，并在实施阶段（由 Plan 第 1 步）锁定，一旦选定不得在 V1 周期内切换。选型评分维度：

| 维度 | TOML | YAML | JSON |
|------|------|------|------|
| 人类可读 | 高 | 高 | 中 |
| 手写友好 | 高 | 高 | 低（引号/逗号繁琐） |
| 解析器体积 | 小（toml++ ~200KB） | 大（yaml-cpp ~1MB） | Qt 自带 |
| 错误信息友好 | 中 | 差（缩进陷阱多） | 好 |
| 版本管理（git diff） | 好 | 好 | 中 |
| 注释支持 | 有 | 有 | 无 |
| Qt 生态 | 需引入第三方 | 需引入第三方 | `QJsonDocument` 内建 |

**推荐 TOML**：手写体验最好、注释支持、Qt 生态外体积可控；退路 JSON（可 zero-dep）。YAML 因缩进陷阱、解析库体积大暂不推荐。

### [INV-2] Schema 版本字段

主题文件**必须**包含顶层 `version` 字段（字符串，语义化版本）。
- 应用内置 `kCurrentSchemaVersion = "1.0"`
- 加载时主次版本号必须与当前兼容（主版本相同，次版本 ≤ 当前）
- 若 `version` 缺失 → 视为语法错误，fallback 到内置默认主题
- 若主版本不兼容 → 弹窗提示升级应用，fallback 到默认
- 保留字段：后续版本可新增字段，旧版本读到未识别字段应忽略但不报错（向前兼容）

### [INV-3] 加载失败必须 fallback 且可见

主题文件任何阶段加载失败（文件不存在 / 语法错误 / schema 错误 / 色值非法）时：
- **必须**fallback 到内置默认主题（跟随系统 → light 或 dark，从未加载过则 light）
- **必须**在 UI 上提示错误（首选状态栏气泡 / 状态栏消息 3s；严重错误用弹窗）
- **禁止**崩溃、禁止静默失败
- 错误信息必须包含文件路径、出错行号（若解析器支持）、出错字段名

### [INV-4] 必填 vs 可选字段

Schema 中每个字段分三类：
- **必填**：如 `version`、`name`、`mode`（light / dark）→ 缺失即视为 schema 错误
- **可选-继承**：未提供时从 `extends` 指向的 base 主题继承（见 INV-6）
- **可选-默认值**：未提供且无继承时使用内置默认值

用户可以只写几行（如只定义 `editorBackground`）即可得到可用主题，其他字段自动从 base 继承或走默认。

### [INV-5] 主题文件存放路径

| 平台 | 路径 |
|------|------|
| Windows | `%APPDATA%/SimpleMarkdown/themes/` |
| Linux | `$XDG_CONFIG_HOME/SimpleMarkdown/themes/` 或 `~/.config/SimpleMarkdown/themes/` |
| macOS（未来） | `~/Library/Application Support/SimpleMarkdown/themes/` |

- 应用启动时**自动扫描**该目录下所有 `*.{toml|yaml|json}`
- 文件名（不含扩展名）作为主题 ID 的一部分，与显示名（文件内 `name` 字段）解耦
- 目录不存在时首次启动自动创建并写入一个示例主题

### [INV-6] 主题继承（extends）

用户主题可以通过顶层 `extends` 字段指定父主题：

```toml
version = "1.0"
name = "我的深色护眼"
extends = "builtin:dark"   # 或 "user:other-theme-file"
[colors]
editorBackground = "#1a1d2e"
```

- 仅覆盖 `extends` 中不同的字段，其他全部继承
- `builtin:light` / `builtin:dark` 作为兜底 base，始终可引用
- **不允许**继承链出现循环（加载时检测，发现环即视为 schema 错误）
- 最大继承深度：5 层（防止恶意深链）

### [INV-7] 切换主题立即生效

切换主题时必须触发 `30-主题系统.md INV-3`：所有 widget 在同一事件循环中 `update()`，不得出现半旧半新状态。

### [INV-8] 合法性验证

加载时必须校验以下：
- 颜色字段：`#RGB` / `#RRGGBB` / `#RRGGBBAA` / `rgb(r,g,b)` / `rgba(r,g,b,a)`，非法即报错
- 数字字段：类型正确（int / float）且在允许范围内（如边框宽度 0–20px，圆角 0–50px）
- 枚举字段：如 `mode ∈ {light, dark}`、`scrollbar.style ∈ {overlay, classic}` 等
- 字体族：不做存在性验证（避免环境差异），但字符串合法即可
- **不允许**抛异常到 UI 线程；所有错误被 `ThemeLoader::Result` 承接

### [INV-9] 可定制字段覆盖范围

V1 必须支持的可定制类别（字段总数 ≥ 80）：

**颜色（colors.*）**：
- 编辑器：`editorBackground` / `editorForeground` / `editorCursor` / `editorSelection` / `editorCurrentLine` / `editorLineNumber` / `editorLineNumberActive` / `editorGutterBackground` / `editorSearchMatch` / `editorSearchMatchCurrent` / `editorPreeditBg` / `editorGutterLine`
- 语法高亮：`syntaxHeading` / `syntaxCode` / `syntaxCodeBg` / `syntaxCodeBlock` / `syntaxCodeBlockBg` / `syntaxLink` / `syntaxList` / `syntaxBlockQuote` / `syntaxFence` / `syntaxFenceBg`
- 预览：`previewBackground` / `previewForeground` / `previewHeading` / `previewLink` / `previewCodeBg` / `previewCodeFg` / `previewCodeBorder` / `previewBlockquoteBorder` / `previewBlockquoteBg` / `previewTableBorder` / `previewTableHeaderBg` / `previewHr` / `previewHeadingSeparator`
- 图片占位 / 错误：`imagePlaceholderBg` / `imagePlaceholderBorder` / `imagePlaceholderText` / `imageErrorBg` / `imageErrorBorder` / `imageErrorText` / `imageInfoText`
- 应用级：`windowBackground` / `menuBackground` / `menuForeground` / `menuHoverBackground` / `tabBackground` / `tabActiveBackground` / `tabForeground` / `tabActiveForeground` / `toolbarBackground` / `statusBarBackground` / `statusBarForeground` / `dialogBackground` / `dialogForeground` / `buttonBackground` / `buttonHoverBackground` / `inputBackground` / `inputBorder`
- 标记 / 选区：`markingHighlight` / `markingHighlightToc` / `selectionBackground`
- 滚动条：`scrollbarThumb` / `scrollbarThumbHover` / `scrollbarTrack`
- 分隔线：`separatorColor`

**字体（fonts.*）**：
- `editor.family` / `editor.sizeDelta`（相对基准的偏移，pt）
- `preview.family` / `preview.sizeDelta`
- `mono.family`（等宽字体，代码块/内联代码）
- `ui.family`（菜单/对话框）
- `preview.headingSizes[1..6]`（h1..h6 倍率）
- `preview.codeFontSizeDelta` / `preview.inlineCodeSizeDelta`

**间距（spacing.*）**：
- `line` / `paragraph` / `heading` / `codeBlock.padding` / `blockquote.padding` / `list.indent` / `table.cellPadding`

**边框（borders.*）**：
- `codeBlock.width` / `codeBlock.radius`
- `blockquote.width` / `blockquote.radius`
- `table.width` / `table.radius`
- `window.radius` / `dialog.radius` / `button.radius` / `input.radius`

**滚动条（scrollbar.*）**：
- `width` / `style`（overlay | classic）/ `radius`

**其他（misc.*）**：
- `focusRing.color` / `focusRing.width`
- `shadow.elevation`（整数 0–4）

V1 未实现的字段允许声明但忽略（为未来保留）。

### [INV-10] 导入 / 导出

- 菜单"设置 → 主题 → 导入主题文件…" → `QFileDialog`，复制到 `themes/` 并重新扫描
- 菜单"设置 → 主题 → 导出当前主题…" → 把当前 `Theme`（含继承展开结果）序列化为所选格式
- 导出的文件必须能被同一应用重新导入且结果等价（round-trip 保真）

### [INV-11] 禁止任意代码执行

主题文件**必须**是纯数据（TOML / YAML-safe / JSON），**不允许**：
- 执行 JavaScript / Lua / Python 代码段
- 加载外部 DLL / so / dylib
- 文件路径字段指向任意文件系统位置（图标资源等若 V2 引入，必须限定在主题自己的目录下）
- YAML 若选用，必须用 safe-loader，禁用自定义 tag（`!!python/object` 等）

### [INV-12] 主题文件纯文本可编辑

- 不使用二进制格式（如 SQLite、binary plist）
- 不使用压缩（主题文件本身 < 20KB，无需压缩）
- 文件编码：**UTF-8 无 BOM**
- 换行符：LF 或 CRLF 均接受，导出时统一为 LF

### [INV-13] 字段命名语义化

- 字段名必须反映**用途**而非**颜色值**（`editorBackground` 而非 `gray100`）
- 分组用分层（`colors.editor.background` 或 `[colors.editor] background = "..."`）而非扁平长名
- 命名风格统一：TOML 用 `snake_case` 或 `camelCase`（实施阶段选定并在文档中固定），字段名保留字列在 INV-9

### [INV-14] 主题市场预留

V1 不实现主题市场（在线下载），但 schema 必须留出空间：
- 顶层可选字段：`id`（全局唯一）、`author`、`homepage`、`license`、`description`、`preview_screenshot`
- 这些字段加载时只展示，不参与渲染

## 4. 接口

### 4.1 `ThemeLoader.h`

```cpp
namespace core {

struct ThemeLoadError {
    QString file;
    int line = -1;          // 解析器能提供时填入
    QString field;          // 出错字段路径（如 "colors.editorBackground"）
    QString message;        // 人类可读原因
};

struct ThemeLoadResult {
    bool ok = false;
    Theme theme;                  // ok 为 false 时是 fallback
    QList<ThemeLoadError> errors; // 所有收集到的错误（验证器非 fail-fast）
    QString sourcePath;           // 加载来源文件
};

class ThemeLoader {
public:
    // 从指定文件加载
    static ThemeLoadResult loadFromFile(const QString& path);

    // 从字符串加载（方便测试和导入预览）
    static ThemeLoadResult loadFromString(const QString& content, const QString& format);

    // 序列化当前 Theme 为字符串（用于导出）
    static QString serialize(const Theme& theme, const QString& format);

    // 扫描用户主题目录，返回元信息列表（不加载完整 Theme）
    struct ThemeInfo {
        QString id;
        QString displayName;
        QString filePath;
        QString mode;  // "light" | "dark"
        QString author;
    };
    static QList<ThemeInfo> scanUserThemes();
    static QList<ThemeInfo> listBuiltinThemes();

    // 当前 schema 版本常量
    static constexpr const char* kSchemaVersion = "1.0";
};

} // namespace core
```

### 5.2 加载流程

```
scanUserThemes()
   ↓
为每个文件调用 loadFromFile(path)
   ↓
1. 读取文件（UTF-8，检测 BOM）
   ↓
2. 解析（TOML / YAML / JSON 解析器）
   → 语法错误：收集到 errors[]，fallback Theme::default()
   ↓
3. 读取 version → 兼容性检查
   → 不兼容：错误，fallback
   ↓
4. 读取 extends → 递归加载父主题（环检测 + 深度限制）
   ↓
5. 把当前主题的字段 merge 到父主题上（浅合并 + 覆盖）
   ↓
6. 校验每个字段
   - 色值合法？
   - 数字范围合法？
   - 枚举合法？
   → 单字段错误：记录到 errors[]，该字段回退到默认值
   ↓
7. 构造 Theme 对象
   ↓
ThemeLoadResult{ok=true/false, theme, errors, sourcePath}
```

非 fail-fast：单字段错误不中止加载，尽可能返回部分可用主题 + 错误列表。仅在"文件读不到 / 语法完全错 / 版本不兼容"时整体 fallback。

### 5.3 继承合并

```
resolvedTheme = builtinDefault
for t in extendsChain (根 → 叶):
    for field in t.providedFields:
        resolvedTheme[field] = t[field]
```

`providedFields` 由解析器给出（TOML / YAML 都能区分"字段未写"和"字段写了空值"）。未写的字段不覆盖 parent。

## 6. 性能预算

| 操作 | 目标 |
|------|------|
| 启动时扫描 themes/ 目录（≤ 20 个文件） | < 100 ms |
| 加载单个主题（含继承） | < 20 ms |
| 切换主题（UI 重绘） | 沿用 30-主题系统 INV < 100ms |
| 导入单个文件（复制 + 解析 + 预览） | < 200 ms |

## 7. 验收条件

- [T-1] 加载内置 light 主题，所有 widget 颜色正确，与当前硬编码的 `Theme::light()` 视觉等价
- [T-2] 把 `themes/my-theme.toml` 放到用户目录后重启，该主题出现在菜单中，切换后生效
- [T-3] 人为把主题文件写坏（删 `version`、写错色值），启动不崩溃、弹窗/状态栏提示错误、当前主题回退到 light
- [T-4] 主题只写 10 行（仅覆盖 `editorBackground` 和 `previewBackground`），其余字段通过 `extends = "builtin:dark"` 从 dark 继承
- [T-5] 连续切换同一文件中的两套不同主题 20 次，无残留旧色；内存不增长（无主题对象泄漏）
- [T-6] "导出当前主题"生成的文件，被另一个干净实例导入后得到等价 Theme（对比所有字段相等）
- [T-7] 1x / 1.25x / 1.5x DPI 下颜色、字号、边框圆角、滚动条宽度均正确响应主题
- [T-8] 主题文件可在 VSCode / Notepad++ 中阅读和编辑，语法高亮正常（TOML/YAML/JSON 任一）
- [T-9] 继承链：A extends B extends builtin:dark，A 只覆盖 1 个字段，B 覆盖 3 个字段，最终结果正确
- [T-10] `grep -rn "QColor(\"#" src/` 除 Theme 构造自身和 ThemeLoader 的默认值常量外无硬编码
- [T-11] 循环继承（A extends B, B extends A）被检测并报错，不死循环
- [T-12] 恶意 YAML（若选用）如 `!!python/object` 被拒绝；或者选 TOML/JSON 从根源规避

## 8. 已知陷阱

### 8.1 解析库选型影响二进制大小

- `toml++`（header-only，~200KB 源码）：推荐
- `yaml-cpp`（编译后 ~1MB）：慎用
- Qt 自带 JSON：零成本，但手写体验差
- **陷阱**：如果引入 yaml-cpp，release 安装包可能增加 1–2MB

### 8.2 YAML 缩进陷阱

若选用 YAML，用户很容易写错缩进（tab vs 空格，块级 vs 流式），错误信息对新手不友好。**强烈建议 TOML** 或 **JSON**。

### 8.3 颜色字段合法性

- `#FFF` / `#FFFFFF` / `#FFFFFFFF` / `rgb(255,255,255)` / `rgba(255,255,255,0.5)` 必须都支持
- `#GGG` 这种字母超出 hex 范围必须拒绝
- `Qt` 的 `QColor(QString)` 对非法字符串返回 invalid 但**不抛异常**，需显式 `isValid()` 校验

### 8.4 路径资源与安全

V1 不允许主题文件引用外部文件（图标、背景图片等）。若 V2 引入，必须：
- 路径相对于主题文件所在目录
- 禁止 `..` / 绝对路径 / URL
- 大小限制（单资源 ≤ 500KB）

### 8.5 热重载 vs 重启加载

V1 采用**重启加载**（启动时扫描一次，或菜单"重新扫描主题"手动刷新），**不实现**文件系统监听热重载。理由：
- 文件写一半被读取的竞态处理复杂
- 性能不是瓶颈（用户切换主题频率低）
- 实施成本高于收益

V2 若需热重载，可考虑 `QFileSystemWatcher`，但要加 debounce（写入完成 500ms 后再读）。

### 8.6 向前兼容

旧版本应用读到新版本 schema 的未知字段时，**必须忽略而不是报错**。否则用户升级应用后，之前能用的主题会突然报错。

### 8.7 弹窗主题跟随

新主题系统不得违反 `30-主题系统.md INV-4/INV-5`：所有 Dialog 仍须 reads-from-Theme，不得硬编码色值。主题插件系统只是 Theme 的**来源**换了，Theme 的**消费方**不变。

### 8.8 默认值同步

`ThemeLoader` 内建的"字段缺失时的默认值"必须与 `Theme::light()` / `Theme::dark()` 的硬编码值保持一致（或者让 light/dark 也走 loader 从内置 TOML 加载，彻底消除二份真相）。推荐后者，作为 INV-1 唯一数据源的体现。

### 8.9 TOC 与标记色（30-主题系统 INV-4 交互）

主题标记高亮色（`markingHighlight` / `markingHighlightToc`）在深浅主题下可见度不同。用户自定义主题必须同时提供两个字段，或由 loader 根据 `mode` 自动派生一个。

### 8.10 UI 反馈低优先级

加载错误不能打断应用启动。启动时若 themes/ 目录有坏文件，应：
- 启动照常完成
- 在状态栏展示"3 个主题加载失败，点击查看详情"
- 点击后弹详情对话框
