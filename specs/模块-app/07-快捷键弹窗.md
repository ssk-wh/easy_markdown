# 快捷键弹窗（ShortcutsDialog）

## 1. 目的

在一个独立的模态弹窗中展示应用全部快捷键清单，支持按关键字过滤。帮助用户
快速查询"某个操作对应哪个快捷键"以及"某个快捷键做什么"，无需进入设置页。

## 2. 输入 / 输出

- **输入**：
  - 用户通过 `Help → Keyboard Shortcuts` 菜单项触发打开
  - 搜索框文本（用于实时过滤）
  - 当前主题 `Theme`（由 `MainWindow` 传入）
- **输出**：
  - 一个 700x500 的模态 `QDialog`，包含标题栏、搜索框、快捷键表格、关闭按钮
  - 表格列：Category（分类）/ Action（操作）/ Shortcut（组合键）

## 3. 行为契约（不变量）

### [INV-1] 跟随主题

弹窗必须响应当前应用主题（亮色 / 深色），遵守 `横切关注点/30-主题系统.md#INV-4`：

- 构造函数接受 `const Theme&`，禁止使用 `Theme::light()` 作为默认值
- 所有颜色通过 `m_theme` 读取，**禁止硬编码十六进制或 RGB 值**
- 弹窗根 `QDialog` 设置 `Qt::WA_StyledBackground`，保证样式表的 `background` 生效
- 暴露 `setTheme(const Theme&)` 支持运行时切换（即使当前只在构造时调用一次）

### [INV-2] 全部控件背景一致

深色主题下必须保证以下元素都是深色：

- 窗口背景
- `QLabel`（包括 "Search:" 文字）
- `QLineEdit`（搜索框）**及其内部显示区**
- `QTableWidget`（快捷键表格主体）**及其 viewport**（行内容区域）
- `QHeaderView`（表头）
- `QPushButton`（关闭按钮）
- 滚动条、表格网格线

**特别注意**：`QTableWidget` 的表格 body 和 `QLineEdit` 的输入区都是
`QAbstractScrollArea` 的 viewport，它们的背景默认用 `QPalette::Base` 绘制。
单靠 stylesheet `QTableWidget { background: ... }` **只会改 frame，内部 viewport
仍是白色**。必须同时 `setPalette` 把 `Base`/`AlternateBase`/`Text` 设成深色。

详见 `横切关注点/30-主题系统.md#INV-5`。

不得出现"表格 frame 是深色但内容区（行）是白色"之类的半套皮肤。

### [INV-3] 过滤实时生效

用户在搜索框输入时，表格立即按 Category / Action / Shortcut 三列做
不区分大小写的子串匹配，并隐藏不匹配的行。搜索框清空时恢复全部行。

### [INV-4] 默认按钮

`Close` 按钮必须设置 `setDefault(true)`，按 Enter 可直接关闭弹窗。

### [INV-5] 全部用户文本必须走 tr()

弹窗内**所有**面向用户的字符串字面量必须用 `tr("...")` 包装，包括但不限于：

- `setWindowTitle("...")` 的标题
- `QLabel`、`QPushButton`、`QLineEdit::setPlaceholderText` 的文案
- `setHorizontalHeaderLabels({...})` 的列名
- 内部数据表（如 `{category, action, shortcut}` 列表）中的分类名和操作描述
- ToolTip / StatusTip / AccessibleName

详见 `横切关注点/60-国际化.md#INV-2`。

`translations/simple_markdown_zh_CN.ts` 中的所有对应条目必须有 `<translation>` 内容，
不得停留在 `type="unfinished"`，否则 Qt 会回落显示英文源串。

## 4. 接口（API 签名）

```cpp
class ShortcutsDialog : public QDialog {
    Q_OBJECT
public:
    // 构造时必须传 theme，否则编译报错（无默认参数）
    explicit ShortcutsDialog(const Theme& theme, QWidget* parent = nullptr);
    void setTheme(const Theme& theme);

private:
    void setupUI();
    void populateShortcuts();
    void applyTheme();
    void filterShortcuts(const QString& text);

    QLineEdit*    m_searchEdit;
    QTableWidget* m_table;
    Theme         m_theme;

    struct ShortcutInfo {
        QString category;
        QString action;
        QString shortcut;
    };
    QVector<ShortcutInfo> m_shortcuts;
};
```

`MainWindow::onShowShortcuts` 调用方式：

```cpp
void MainWindow::onShowShortcuts() {
    ShortcutsDialog dialog(m_currentTheme, this);
    dialog.exec();
}
```

## 5. 核心算法

### 5.1 构造 → 渲染流程

```
ShortcutsDialog(theme, parent)
  ├─ setAttribute(WA_StyledBackground, true)
  ├─ m_theme = theme
  ├─ setupUI()            // 创建控件，不碰颜色
  ├─ populateShortcuts()  // 填充静态快捷键清单
  └─ applyTheme()         // 根据 m_theme 一次性刷 stylesheet
```

### 5.2 颜色来源（复用 Theme 现有字段）

| 用途 | Theme 字段 |
|------|-----------|
| 窗口背景 | `previewBg` |
| 文字颜色 | `previewFg` |
| 搜索框背景 | `editorBg` |
| 搜索框边框 | `previewCodeBorder` |
| 表格背景 | `previewBg` |
| 表格交替行背景 | `previewCodeBg` |
| 表头背景 | `previewTableHeaderBg` |
| 按钮背景 | `editorBg` |
| 按钮 hover 背景 | `previewTableHeaderBg` |
| 网格/边框线 | `previewTableBorder` |

这些字段在 `Theme::light()` / `Theme::dark()` 中已有明确值，无需新增 Theme 字段。

### 5.3 过滤算法

```
for each row i:
    if filter is empty → show row
    else:
        for each column c in [category, action, shortcut]:
            if row[i][c].toLower().contains(filter) → show row; break
        else → hide row
```

## 6. 性能预算

- 打开弹窗 < 50 ms
- 过滤 1 次输入 < 5 ms（当前约 25 条快捷键）

## 7. 验收条件（测试用例清单）

- **[T-1]** 亮色主题下打开：窗口白底深字，表格清晰可读
- **[T-2]** 深色主题下打开：窗口、搜索框、表格**body（viewport）**、
  表头、按钮、滚动条**全部**为深色，**不得出现表格内容区是白色的情况**
- **[T-3]** 亮色主题下构造、`setTheme(dark)`、`show()`：应立即呈现深色
- **[T-4]** 搜索框输入 "save"：隐藏所有非 "save" 相关行
- **[T-5]** 搜索框清空：恢复所有行
- **[T-6]** 按 Enter 关闭弹窗
- **[T-7]** `grep -rn "QColor(\"#" src/app/ShortcutsDialog.cpp` 无结果
- **[T-8]** 从菜单多次打开关闭，无内存泄漏（RAII 由栈对象保证）
- **[T-9]** 截屏对比验证：
  1. 亮色主题下打开弹窗，截屏保存为 `tmp/shortcuts_light.png`
  2. 切换深色主题，再次打开弹窗，截屏保存为 `tmp/shortcuts_dark.png`
  3. 两张图同一位置（比如表格第 5 行中央）的像素颜色必须有显著差异
     （亮色应 >= `#f0f0f0`，深色应 <= `#404040`）

## 8. 已知陷阱

### 8.1 硬编码颜色

早期实现中 `applyTheme()` 直接写 `QColor(45, 45, 45)` 等 RGB 值，违反
`横切关注点/30-主题系统.md#INV-1`。修复时全部替换为 `m_theme.xxx`。

### 8.1.1 QTableWidget viewport 白底（首次踩坑于 2026-04-13）

只用 stylesheet 设置 `QTableWidget { background: ... }` 后，外层 frame 变深
但表格内容区（行）仍是纯白色。原因是 `QAbstractScrollArea::viewport()` 的
背景用 `QPalette::Base` 绘制，stylesheet 的 background 属性只影响 frame。

**必须**：

```cpp
QPalette pal = palette();
pal.setColor(QPalette::Base, m_theme.previewBg);
pal.setColor(QPalette::AlternateBase, m_theme.previewCodeBg);
pal.setColor(QPalette::Text, m_theme.previewFg);
setPalette(pal);
m_table->setPalette(pal);
m_table->viewport()->setPalette(pal);  // 关键：viewport 也要设
```

同样适用于 `QLineEdit`（也是 QAbstractScrollArea 子类的变体）。

### 8.1.2 QString::arg 超过 9 个占位符的 bug

`QString(s).arg(a,b,c,d,e,f,g,h,i).arg(j)` 看似可行，但如果 stylesheet 里
有 `%10` 占位符，Qt 会把 `%10` 解析为 `%1` + 字面字符 "0"，导致 selBg 被
插入到其他位置而非预期。应把所有占位符严格控制在 `%1..%9` 内，重复使用。

### 8.2 QDialog 背景未变

Qt 5 下 `QDialog { background: #2b2b2b; }` 在未调用
`setAttribute(Qt::WA_StyledBackground, true)` 时**不生效**，只有 `QLabel` 等
子控件的样式生效，导致"文字是灰的但窗口是白的"。必须显式开启 WA_StyledBackground。

### 8.3 默认 Theme::light() 误导

构造函数用 `m_theme(Theme::light())` + 外部 `setTheme` 的两段式，调试时
容易错把"外部没调 setTheme"当作"样式表不生效"。强制构造注入主题可以
把错误前移到编译期。

### 8.4 全局 stylesheet 与局部 stylesheet 冲突

`MainWindow` 在 `applyTheme` 时会给自己 `setStyleSheet`，其规则
`QDialog { ... }`、`QDialog QLabel { ... }` 会继承到 parent 为 MainWindow
的弹窗。但**弹窗一旦自行 `setStyleSheet`**，自身的样式表会覆盖父级传递的
部分规则（Qt 样式表不是简单的 CSS 级联）。因此弹窗自管样式时，
必须把所有子 widget 的规则（QLineEdit / QTableWidget / QHeaderView / QPushButton）
都写进来，不能只写 QDialog。
