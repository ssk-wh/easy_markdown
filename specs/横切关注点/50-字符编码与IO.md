# 字符编码与文件 IO

## 1. 目的

定义源代码编码、文件读写编码、换行符处理、字符索引单位等 IO 相关规则。

## 2. 输入 / 输出

- **输入**：用户磁盘上的 `.md` / `.markdown` / `.txt` 文件
- **输出**：`QString`（UTF-16）内存表示

## 3. 行为契约

### [INV-1] 源代码编码

- 所有 C++ / Markdown / CMake 源文件统一使用 **UTF-8 无 BOM**（项目现状）
- 修改已有源文件时**严禁**变更 BOM 状态（不要给 NO-BOM 文件加 BOM，也不要删除历史遗留的 BOM）
- 若发现旧文件是 GBK，需要先转成 NO-BOM UTF-8 再编辑：
  ```python
  with open(path, 'rb') as f:
      text = f.read().decode('gbk')
  with open(path, 'wb') as f:
      f.write(text.encode('utf-8'))  # 不写 BOM
  ```

### [INV-2] 文件读取

- 打开文件时按以下顺序探测编码：
  1. 检测 BOM（UTF-8 BOM / UTF-16 LE BOM / UTF-16 BE BOM）
  2. 若无 BOM，默认按 UTF-8 读
  3. UTF-8 解码失败时回退到系统默认编码
- 记录原始编码，保存时用同一编码

### [INV-3] 文件保存

- 保留原始 BOM（有则保留，无则不加）
- 保留原始换行符（CRLF / LF）
- 新建文件使用平台默认换行符，不写 BOM（除非用户显式要求）

### [INV-4] 字符索引单位

- 内部所有文本索引以 **UTF-16 码元** 为单位（与 QString 一致）
- 对外 API（如搜索结果偏移）也使用 UTF-16 码元
- 涉及字符（code point）计数时必须显式转换，不得直接用字节/码元数

### [INV-5] 大文件处理

- 文件 > 10 MB 时使用 `MappedFile`（mmap）只读映射
- 超大文件（> 100 MB）只进入阅读模式，不允许编辑

### [INV-6] 外部修改检测

- 打开文件后监听 `QFileSystemWatcher`
- 外部修改时弹窗提示，用户选择重载或保留内存版本
- 重载时保留光标位置和选区

## 4. 接口

```cpp
class Document {
public:
    // 读取文件到内存，自动检测编码与换行符
    bool loadFromFile(const QString& path, QString* errorOut = nullptr);

    // 保存到文件，使用记录的编码与换行符
    bool saveToFile(const QString& path, QString* errorOut = nullptr);

    // 文件编码枚举
    enum class Encoding { Utf8, Utf8WithBom, Utf16Le, Utf16Be, System };
    Encoding fileEncoding() const;

    // 换行符风格
    enum class LineEnding { Lf, Crlf, Mixed };
    LineEnding lineEnding() const;
};
```

## 5. 核心算法

### 编码检测流程

```
读取前 4 字节
  │
  ├─ == 0xEF 0xBB 0xBF    → UTF-8 BOM
  ├─ == 0xFF 0xFE         → UTF-16 LE BOM
  ├─ == 0xFE 0xFF         → UTF-16 BE BOM
  └─ 其他
       │
       ▼
     尝试 UTF-8 解码
       │
       ├─ 成功 → UTF-8（无 BOM）
       └─ 失败 → 系统默认编码
```

### 换行符检测

```
统计 \r\n 和 \n 出现次数：
  - 全部 \r\n      → CRLF
  - 全部 \n        → LF
  - 两者都有       → Mixed，按多数决
```

## 6. 性能预算

- 加载 1 MB 文件 < 50 ms
- 加载 10 MB 文件（mmap）< 100 ms
- 编码检测 < 5 ms

## 7. 验收条件

- [T-1] 加载 UTF-8 BOM 文件不丢 BOM
- [T-2] 加载 GBK 文件显示乱码的 fallback 到系统编码
- [T-3] 保存后文件大小与原文件相当（不意外增减 BOM）
- [T-4] 中文字符搜索结果偏移正确（不按字节计算）
- [T-5] 外部修改 Reload 后光标位置合理

## 8. 已知陷阱

- **QString::fromUtf8 不检测 BOM**：必须手动检测和剥离
- **Windows `notepad` 默认 CRLF**：保存时不要强制改成 LF，否则与系统工具不兼容
- **混合换行符**：实际会出现，按多数决处理，给用户选项统一化
