# 线程模型

## 1. 目的

定义 SimpleMarkdown 的线程边界和跨线程通信规则，防止数据竞争、死锁、主线程卡顿。

## 2. 输入 / 输出

N/A（架构级约束）

## 3. 行为契约

### [INV-1] 主线程独占 GUI

- 所有 `QWidget` 方法调用必须在主线程
- 所有 `QPainter` 调用必须在主线程
- 所有 `QPaintDevice` 相关操作必须在主线程

### [INV-2] 后台线程清单

只允许以下后台线程存在：

| 线程 | 职责 | 源码 |
|------|------|------|
| 解析线程池 | cmark-gfm 解析 | `ParseScheduler` |
| 搜索线程 | 搜索替换匹配 | `SearchWorker` |
| 图片加载线程 | 本地图片读取与解码 | `ImageCache` |

新增后台线程必须先更新本文档。

### [INV-3] 跨线程数据传递规则

| 数据类型 | 传递方式 |
|---------|---------|
| **不可变快照**（如 `MarkdownAst`） | `std::shared_ptr<const T>` |
| **大块数据**（如 QImage） | 值传递 + 移动语义（Qt 隐式共享） |
| **通知事件** | signal-slot (`Qt::QueuedConnection` 或 `AutoConnection`) |
| **可变共享状态** | **禁止**——必须改为不可变快照或加锁 |

### [INV-4] 跨线程 signal-slot 规则

```cpp
// ✓ 正确：默认 AutoConnection，跨线程时自动变 Queued
connect(worker, &Worker::done, main, &Main::onDone);

// ✗ 禁止：DirectConnection 会在 emit 线程执行 slot
connect(worker, &Worker::done, main, &Main::onDone, Qt::DirectConnection);
```

### [INV-5] 后台线程生命周期

- 后台线程必须有明确的启动/停止点
- 停止时必须等待线程退出（`QThread::wait()`）才能析构其数据
- 应用退出时先停止所有后台线程，再销毁主窗口

### [INV-6] AST 不可变

`MarkdownAst` 在构造完成后不得修改。传递给主线程后，主线程只读。若需要新版本，重新解析生成新 AST。

### [INV-7] 主线程不得阻塞

主线程单次事件处理 < 16 ms。耗时操作必须：
1. 移入后台线程，或
2. 分片处理（用 `QTimer::singleShot(0, ...)` 延续）

## 4. 接口

### ParseScheduler

```cpp
class ParseScheduler : public QObject {
    Q_OBJECT
public:
    // 请求解析，自动 30ms 防抖
    void scheduleParse(const QString& source);

signals:
    // 解析完成，AST 已经是不可变快照
    // 连接方式：AutoConnection（默认变 Queued 跨线程）
    void astReady(std::shared_ptr<const MarkdownAst> ast);
};
```

### SearchWorker

```cpp
class SearchWorker : public QObject {
    Q_OBJECT
public slots:
    // 在后台线程执行搜索
    void search(const QString& source, const SearchQuery& q);

signals:
    void resultReady(std::vector<SearchMatch> matches);
};
```

## 5. 核心算法

### 解析流程（跨线程）

```
主线程                             解析线程
───────                            ────────
EditorInput::keyPress
  │
  ▼
Document::insert()
  │
  ▼
ParseScheduler::scheduleParse()
  │ (debounce 30ms)
  │
  │ QThreadPool::start(task)
  │─────────────────────────────▶ run()
  │                                 │
  │                                 ▼
  │                               MarkdownParser::parse()
  │                                 │
  │                                 ▼
  │                               std::make_shared<const MarkdownAst>(...)
  │                                 │
  │ ◀──────────────────────────── emit astReady(ast) [Queued]
  │
  ▼
onAstReady(ast)
  │
  ▼
PreviewLayout::layout(*ast, ...)
```

## 6. 性能预算

- 线程切换延迟 < 5 ms
- 主线程事件循环单次 < 16 ms

## 7. 验收条件

- [T-1] ThreadSanitizer 运行无数据竞争警告
- [T-2] 解析大文件（10k 行）时主线程 UI 不卡顿
- [T-3] 应用退出时无"线程未退出"警告
- [T-4] `grep -rn "Qt::DirectConnection" src/` 只在明确同线程场景出现

## 8. 已知陷阱

### 8.1 QSettings 注册表写入阻塞

Windows 下 QSettings 写入会触发注册表 flush，频繁调用会卡顿主线程。解决方案：
- 防抖合并写入
- 或者在主线程 idle 时批量写

### 8.2 shared_ptr 控制块竞争

虽然 shared_ptr 本身线程安全，但 deref 指向的对象时，对象必须是 immutable 或加锁。AST 是 immutable 所以无需锁。

### 8.3 emit 时 receiver 已销毁

跨线程 queued emit 时，receiver 可能在事件到达前被销毁。必须：
- 使用 QPointer 或 WeakPointer 监控
- 或者在 receiver 析构时先 disconnect
