# 性能预算

## 1. 目的

集中管理所有性能指标，作为 PR 审核和回归测试的量化基线。任何性能相关改动必须先更新本文。

## 2. 输入 / 输出

N/A（指标集合）

## 3. 行为契约

### [INV-1] 启动性能

| 指标 | 目标 | 基准机器 |
|------|------|---------|
| 冷启动到首窗口显示 | < 500 ms | i5-8250U + SSD |
| 首次 Tab 内容就绪 | < 200 ms | 同上 |
| 空载内存占用 | < 30 MB | 同上 |

### [INV-2] 编辑性能

| 指标 | 目标 |
|------|------|
| 单次按键到屏幕更新 | < 3 ms |
| 滚动一屏 | < 10 ms |
| 全文档语法高亮 | < 200 ms |
| 撤销/重做 | < 5 ms |
| 查找（1k 行文件） | < 50 ms |

### [INV-3] 预览性能

| 指标 | 目标 |
|------|------|
| 预览更新（含 30ms 防抖） | < 80 ms |
| 滚动一屏 | < 16 ms（60fps） |
| 本地图片加载 | < 30 ms |
| 代码块高亮 | < 10 ms/块 |

### [INV-4] 解析性能

| 指标 | 目标 |
|------|------|
| 1k 行 Markdown 解析 | < 10 ms |
| 10k 行 Markdown 解析 | < 50 ms |
| 防抖延迟 | 30 ms |

### [INV-5] 文件 IO 性能

| 指标 | 目标 |
|------|------|
| 加载 1 MB 文件 | < 50 ms |
| 加载 10 MB 文件（mmap） | < 100 ms |
| 保存文件 | < 50 ms |

### [INV-6] 主线程不阻塞

任何主线程事件处理单次 < 16 ms（保证 60fps）。超过必须：
- 移到后台线程
- 或分片处理

### [INV-7] 性能回归红线

PR 中出现以下情况**必须拦截**：
- 任何 INV-1~5 指标劣化超过 20%
- 主线程出现 > 16 ms 阻塞
- 内存占用超过基线 + 5 MB

## 5. 核心算法

### 性能测量方法

```cpp
// 使用 QElapsedTimer 采样
QElapsedTimer t;
t.start();
doWork();
qint64 ns = t.nsecsElapsed();
// 上报到 benchmark 或日志
```

### 基准测试

基准测试位于 `tests/benchmark/`，CI 每次构建自动运行。

## 7. 验收条件

- [T-1] 所有 INV-1~5 指标有对应的自动化基准测试
- [T-2] CI 中对比 baseline，劣化 > 20% 自动标红
- [T-3] 主线程长任务探针：主线程消息循环空转时间占比 > 70%

## 8. 已知陷阱

- **Debug 构建不代表 Release 性能**：测试必须在 Release 模式
- **第一次运行包含冷启动开销**：取第 2-N 次运行的中位数
- **QPainter 抗锯齿开销**：某些绘制操作开启 AA 后慢 2-3x，非必要不开
