# 主题系统

## 1. 目的

定义亮色/暗色主题的切换机制、颜色变量集合、以及所有 UI 模块如何响应主题变化。

## 2. 输入 / 输出

- **输入**：用户主题选择（亮色 / 暗色 / 跟随系统）
- **输出**：统一的 `Theme` 结构，供所有 UI 模块读取

## 3. 行为契约

### [INV-1] 唯一数据源

- 全局只有一个 `Theme` 实例（由 `core::Theme` 管理）
- 所有 UI 模块通过同一份 `Theme` 读取颜色
- 禁止在单个模块中硬编码颜色

### [INV-2] 主题结构必须分层

`Theme` 分成语义层（semantic）和原始层（raw）：

- **语义层**：`editorBackground`、`codeBlockBackground`、`linkColor` 等有明确用途的颜色
- **原始层**：`color.gray[100]`、`color.blue[500]` 等调色板

UI 代码只允许读语义层，不读原始层。

### [INV-3] 切换主题时全部 widget 必须响应

切换主题后：
- 所有 `EditorWidget` 重新 `update()`
- 所有 `PreviewWidget` 清空块缓存后重绘
- 菜单栏、Tab 栏、滚动条、弹窗 **全部**跟随

### [INV-4] 深色模式下弹窗跟随

Windows 下 Qt 5.15 的 QDialog 默认不响应深色主题。所有 `QDialog`/`QMessageBox` 子类必须满足以下至少一条：

1. **优先方案**：弹窗**不自行调用 `setStyleSheet`**，依赖 `MainWindow` 的全局 stylesheet
   传导（Qt 的 `QDialog { ... }` / `QDialog QLabel { ... }` 选择器会作用于 parent 为
   `MainWindow` 的弹窗）。`ChangelogDialog` 采用此方案。
2. **备选方案**：弹窗自行管理样式时，必须满足：
   - 构造函数接受 `const Theme&` 参数，不得默认用 `Theme::light()`
   - `setStyleSheet` 中所有颜色**从 `Theme` 结构读取**，**禁止硬编码十六进制或 RGB**
   - 对 `QDialog` 根容器调用 `setAttribute(Qt::WA_StyledBackground, true)`，
     否则样式表的 `QDialog { background: ... }` 规则不会覆盖 palette 绘制的背景
   - 弹窗必须暴露 `setTheme(const Theme&)` 方法，支持运行时主题切换

**反模式**：`bool dark = theme.isDark;` 后用 if/else 硬编码两套 `QColor(...)`，
这与 INV-1 "唯一数据源"相违背。

### [INV-5] 容器类控件必须同时设置 palette 和 stylesheet

以下 `QAbstractScrollArea` 子类的 **viewport** 使用 `QPalette::Base` 绘制背景，
**单纯的 stylesheet `QXxx { background: ... }` 只能影响 frame，viewport 仍是
QPalette::Base 的默认白色**：

- `QTableWidget` / `QTableView`
- `QListWidget` / `QListView`
- `QTreeWidget` / `QTreeView`
- `QTextEdit` / `QTextBrowser` / `QPlainTextEdit`
- `QScrollArea`

弹窗若使用这些控件展示深色主题内容，必须同时：

1. **palette**：对弹窗（及关键子控件）调用 `setPalette()`，设置
   `QPalette::Base`、`QPalette::AlternateBase`、`QPalette::Text`、
   `QPalette::Window`、`QPalette::WindowText`、`QPalette::Highlight`、
   `QPalette::HighlightedText` 等角色
2. **stylesheet**：覆盖 `QHeaderView::section`、`gridline-color`、
   `selection-background-color`、`QScrollBar` 等 palette 无法表达的细节

仅设置 palette 会让 header 和 grid line 不跟随；仅设置 stylesheet 会让
viewport 背景保持白色。两者缺一不可。

### [INV-5] 系统主题监听

启用"跟随系统"时，必须监听系统主题变化事件：
- Windows: `QApplication::paletteChanged`
- Linux: 同上（不同桌面环境行为不一）

### [INV-6] 主题数据可从外部配置文件构造

`Theme` 结构必须能从外部配置文件（TOML / YAML / JSON）通过 `core::ThemeLoader`
构造，不得在 `Theme::light()` / `Theme::dark()` 之外的代码里硬编码整套颜色。
这是主题插件系统（`specs/模块-app/12-主题插件系统.md`）的基础不变量：

- `Theme::light()` / `Theme::dark()` 允许硬编码作为"内置主题"的 ground truth，
  也可选择通过 `ThemeLoader` 从 `resources/themes/*.toml` 加载，两种方式等价
- 用户自定义主题**只能**通过 `ThemeLoader` 构造
- UI 代码（editor / preview / app）不得硬编码颜色，只能读 `Theme` 字段
- 新增任意 `Theme` 字段时，必须同步在主题插件系统 schema INV-9 字段清单中声明
  并给出默认值，否则用户主题无法覆盖该字段

## 4. 接口

```cpp
struct Theme {
    // 语义层
    QColor editorBackground;
    QColor editorForeground;
    QColor lineHighlight;
    QColor selection;
    QColor gutterBackground;
    QColor gutterForeground;

    QColor previewBackground;
    QColor previewForeground;
    QColor codeBlockBackground;
    QColor codeBlockBorder;
    QColor inlineCodeBackground;
    QColor inlineCodeForeground;
    QColor blockquoteBorder;
    QColor tableBorder;
    QColor linkColor;
    QColor headingForeground;

    QColor markingHighlight;       // 内容标记的荧光色
    QColor markingHighlightDark;   // 深色模式下的荧光色

    // 主题切换
    static Theme light();
    static Theme dark();
    static Theme fromSystem();
};
```

## 5. 核心算法

### 切换流程

```
用户点击菜单 → ThemeMode::setMode(Dark)
  │
  ▼
emit themeChanged(newTheme)
  │
  ▼
所有 Widget 在 slot 中：
  - 缓存清空
  - update()
```

## 6. 性能预算

- 切换主题 < 100 ms（含所有 widget 重绘）

## 7. 验收条件

- [T-1] 深色模式下所有弹窗（设置/快捷键/更新日志）跟随深色：窗口背景、文字、
  表格、输入框、按钮、表头必须全部为深色，不得出现纯白底或浅灰边框
- [T-2] 切换主题后无残留的旧主题颜色
- [T-3] 跟随系统时，系统主题变化 3 秒内响应
- [T-4] `grep -rn "QColor(\"#" src/app/` 除 Theme 自身外无硬编码颜色
- [T-5] 弹窗在亮色主题下构造、随后 `setTheme(dark)`，能在下次 `exec()` 前完整变暗

## 8. 已知陷阱

- **弹窗不跟随**：早期快捷键弹窗用 QDialog 默认样式，深色模式下字看不清。
  必须按 INV-4 的两个方案之一实现。
- **QDialog 背景样式表不生效**：Qt 5 下 `QDialog { background: ... }` 仅在
  `setAttribute(Qt::WA_StyledBackground, true)` 之后才覆盖 palette 背景。
  忘了加会看到样式表里其他规则（如 QLabel 文字色）生效但窗口背景仍是白色。
- **viewport 白底**（见 INV-5）：对 `QTableWidget` 设置 stylesheet 后表格 frame
  变深但内容区仍是白色，因为 QAbstractScrollArea 的 viewport 用 `QPalette::Base`
  绘制。必须同时 `setPalette` 把 Base / AlternateBase / Text 设成深色。
  2026-04-13 ShortcutsDialog 修复中首次暴露此坑。
- **QString::arg 占位符数量限制**：`QString(...).arg(a, b, c, d, e, f, g, h, i)`
  的单次调用最多处理 9 个占位符，同时 `%10` 会被误识别为 `%1` + "0"。
  超过 9 个值需要用字符串拼接或分多次 arg，或把所有占位符严格限制在 %1..%9 之内。
- **弹窗自行 setStyleSheet 覆盖全局**：`QDialog` 在调用自己的 `setStyleSheet` 时
  会屏蔽从 `MainWindow` 继承的样式，必须一次性把所有子 widget 的规则都写进来
  （输入框、按钮、表格、表头都要覆盖）。
- **默认 Theme::light() 构造**：弹窗成员 `m_theme` 默认亮色、构造函数里先 `applyTheme()`
  再由外部 `setTheme()`，这种两段式在调试时容易把问题当成"没跟随"。推荐构造函数
  直接接收 `const Theme&`。
- **块缓存不清**：PreviewBlockCache 切换主题后如果不清空，会混用两套颜色。
- **深浅标记色差**：同一个荧光色在两种主题下可见度不同，必须提供两套颜色。
