# 状态栏重新布局（左信息 / 右元数据+保存时间）

## 背景

当前状态栏信息混在一起，阅读负担重。mockup `tmp/theme_mocks/mock_a_arctic_frost.html` 给出了清晰的左右分栏方案：编辑动态信息靠左，文件元数据+保存状态靠右，符合用户从左到右"我在写什么 → 这个文件的状态"的阅读路径。

## 动作

- [ ] 读 `tmp/theme_mocks/mock_a_arctic_frost.html` 的 statusbar 结构，抽取视觉规则（间距 / 分隔符 / 字号）
- [ ] 状态栏拆为 **左区（statusLeft）** 与 **右区（statusRight）**，中间 stretch
- [ ] 左区顺序：行 `Ln X` / 列 `Col Y` / 字数 / 字符数 / 阅读时间 `~N 分钟`
- [ ] 右区顺序：编码（UTF-8）/ 换行（LF/CRLF）/ 语言（Markdown）/ 主题名 / **保存状态**
- [ ] 保存状态文本规则：
    - 无磁盘路径 → "未保存（新文件）"
    - 有路径且 dirty → "未保存修改"（警示色）
    - 有路径且 clean → "已保存 · N 分钟前"（根据 `QFileInfo::lastModified` 或内部保存时间戳计算）
- [ ] `QTimer` 每 30s 触发一次"已保存 · X 分钟前"文案刷新（不访问磁盘，只重算相对时间）
- [ ] 视觉跟随 Theme：分隔符 `·` 颜色、主文本色、警示色都走 Theme 字段
- [ ] Spec 更新：`specs/模块-app/` 状态栏章节新增 INV（左右区顺序 / 相对时间刷新 30s）+ T 条目

## 验收

- 打开一个已保存文件，右下显示"已保存 · 0 分钟前"，2 分钟后变为"已保存 · 2 分钟前"
- 编辑后右下立即变"未保存修改"（警示色），保存后回到"已保存 · 0 分钟前"
- 新建未命名 Tab 右下显示"未保存（新文件）"
- 切换主题（浅色↔深色）状态栏所有文本和分隔符配色同步更新

## 风险

- 自动保存 plan 会频繁改变保存时间；必须复用 Document 的保存信号而不是自己轮询磁盘
- 多 Tab 切换时要刷新状态栏绑定到当前 Tab 的 Document
