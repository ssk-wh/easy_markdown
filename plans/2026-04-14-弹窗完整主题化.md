# 弹窗完整主题化（从二元适配升级到主题插件全适配）

## 背景

### 用户反馈（原话）

> 切换主题后，关于界面、快捷键弹窗、更新日志弹窗、欢迎页、文档统计等所有对话框的视觉
> 元素（按钮、文字、背景、边框、表格）需要**全部适配当前主题**，而不是只有深/浅二元适
> 配。用户期望：关于弹窗的按钮 hover 用当前主题的 accent 色（薄荷清新 → 薄荷绿
> `#4ECDC4`，冰霜蓝白 → iOS 蓝 `#0A84FF`），背景色跟 `theme.editorGutterBg` 等等。

### 当前状态（事实）

1. `specs/横切关注点/30-主题系统.md` 的 **INV-4** 原文：
   > "Windows 下 Qt 5.15 的 QDialog 默认不响应深色主题。所有 QDialog/QMessageBox 子类
   > 必须满足：… 深色下弹窗统一灰黑调，不随具体深色主题变化"

   → 该不变量只覆盖 **light ↔ dark 二元切换**，显式允许"所有深色主题下弹窗统一
   `#2b2b2b`"。主题插件系统（12-主题插件系统.md）落地后，多了 6 款内置主题
   （冰霜蓝白 / 琥珀桃暖 / 深夜极光 / 纸面极简 / 薄荷清新 / 墨禅单色），
   此约束已明显不合用户预期。

2. **MainWindow.cpp `applyTheme` 的实际实现**（源文件 840~980 行）：
   - **深色分支（854-932 行）**：弹窗样式硬编码 `#2b2b2b/#3c3f41/#1e1e1e/#555/#ccc`
     （898-909 行），**完全不读 theme.accentColor / theme.editorGutterBg**。
   - **非深色分支（933-997 行）**：根本**没有** `QDialog/QMessageBox/QPushButton` 相关
     stylesheet 段，依赖 Qt 原生 palette。结果是 6 款浅色主题下所有 QMessageBox
     系弹窗视觉完全一样（原生灰白按钮，无 accent 色）。

3. **受影响弹窗的主题化程度**：见下节清单。

## 目标

升级主题系统的弹窗层，使**所有弹窗在任意主题下**：

1. 背景色来自 `theme.editorBg` / `theme.editorGutterBg` 等 Theme 字段
2. 按钮 hover / 选中态使用 `theme.accentColor`
3. 边框、文字色来自 Theme，非硬编码
4. 切换主题时已打开的弹窗能立即响应（而非下次打开才生效）

对应 Spec INV 升级见「Spec 更新」段。

## 步骤拆分

### Step 1：全局 QDialog / QMessageBox 样式从 Theme 派生

- [ ] 在 `MainWindow.cpp` 抽取 `buildDialogStyleSheet(const Theme& t)` 辅助函数
      返回 `QDialog / QMessageBox / QLabel / QTextEdit / QTextBrowser / QPushButton`
      的统一 css，所有颜色来自 Theme 字段：
      - background: `t.editorBg`
      - color: `t.editorFg`
      - QTextEdit bg: `t.editorGutterBg`
      - 边框: `t.editorGutterLine`
      - QPushButton bg: `t.editorGutterBg`
      - QPushButton hover bg: `t.accentColor` with alpha mixed
      - QPushButton pressed bg: `t.accentColor.darker(110)`
      - selection: `t.accentColor`
- [ ] 删除 `MainWindow.cpp` 898-909 行的深色硬编码段，改为调用
      `buildDialogStyleSheet(theme)`
- [ ] 在非深色分支（933-997 行）同样注入该 css（当前缺失）
- [ ] 对每个 `QDialog` / `QMessageBox` 子类，注意触发 `WA_StyledBackground`
      （弹窗自行构造前必须 setAttribute，否则窗口背景仍是 palette 绘制）

### Step 2：ChangelogDialog 完整主题化

- [ ] `ChangelogDialog.h` 新增私有成员 `Theme m_theme;`，构造函数保存
- [ ] `.cpp` 构造函数结尾调用 `setAttribute(Qt::WA_StyledBackground, true)`
- [ ] 新增 `setTheme(const Theme&)` 公有槽：更新 m_theme、
      调用 `m_previewWidget->setTheme(t)`、对自身 `setStyleSheet(...)`（按钮 hover
      accent / 边框 / 关闭按钮背景用 `t.editorGutterBg`）
- [ ] MainWindow 的 `themeChanged` 信号 → 已打开的 ChangelogDialog::setTheme

### Step 3：ShortcutsDialog hover / selection 改用 accent 色

- [ ] `ShortcutsDialog::applyTheme`（第 141~149 行）新增读取 `m_theme.accentColor`
- [ ] css 中的 `QPushButton:hover`、`QPushButton:pressed`、
      `QTableWidget::item:selected` 用 accent 派生色

- [ ] palette 的 `QPalette::Highlight` 从 `editorSelection` 换成 `accentColor`

## 风险与回滚

- **风险 1（Qt 5.15 QMessageBox 样式覆盖不全）**：
  QMessageBox 的 icon、标题字色、button box 布局在某些 Qt 版本不完全响应 stylesheet，
  尤其 Windows 原生 icon (`QStyle::SP_MessageBoxWarning`) 颜色固定
  - 缓解：Step 4 准备方案 A / 方案 B 两条路径；若 A 不足则走 B 包装函数 + 手动
    setIconPixmap(涂色后的)
- **风险 2（WA_StyledBackground 遗漏）**：
  每个 QDialog 都要 setAttribute 一次，遗漏就会看到样式表里其它规则生效但窗口背景
  仍白（Spec 30 已知陷阱复发）
  - 缓解：在 `buildDialogStyleSheet` 的调用处用 checklist 逐一核对
    （ShortcutsDialog / ChangelogDialog / 新包装的 ThemedMessageBox）
- **风险 3（QTableWidget viewport 白底）**：
  ShortcutsDialog 切换到 accent 色的 palette 后，viewport 必须同步 setPalette +
  stylesheet 双管齐下（Spec 30 INV-5），只改一个会出现 header 变色 / 内容区仍白
  - 缓解：保持现有双管齐下结构不变，仅替换具体色值
- **风险 4（运行时主题切换时机）**：
  Qt 对已显示的 QDialog 的 setStyleSheet 重绘有时需要 `ensurePolished()` 或
  `style()->unpolish()/polish()` 才立即生效
  - 缓解：`setTheme` 实现末尾显式调用 `ensurePolished() + update()`，必要时
    `style()->polish(this)`
- **风险 5（accent 作为 hover 色饱和度过高）**：
  冰霜蓝白 accent `#0A84FF` 全饱和做按钮 hover 背景会非常刺眼
  - 缓解：用 `QColor::lighter(140)` 或带 alpha 30% 叠加到 `editorGutterBg` 上
- **回滚方案**：单 commit 提交，出问题 `git revert`；Spec 与代码同一 commit 内

## Spec 更新详细草案

### `specs/横切关注点/30-主题系统.md` INV-4（重写）

```markdown
### [INV-4] 所有弹窗必须跟随当前主题

所有 `QDialog` / `QMessageBox` 子类在**任意主题**（不仅仅是 light/dark）下必须：

1. 构造时接受 `const Theme&` 或在显示前由 MainWindow 注入 Theme
2. 窗口背景、文字、边框、按钮、选中态色值**全部从 Theme 字段读取**
3. 禁止 `if (theme.isDark) { "#2b2b2b" } else { "..." }` 的二元硬编码分支
4. 对 QDialog 根容器 setAttribute(Qt::WA_StyledBackground, true)
```

### 新增 [INV-4a][accent 色体现]

```markdown
弹窗中的按钮 hover、选中行高亮、焦点边框**必须使用 theme.accentColor** 或其派生色
（`lighter(N)` / `darker(N)` / 与背景 alpha 混合）。不同主题下弹窗应能通过 accent
色辨识，而非所有浅色主题按钮长得一样。
```

### 新增 [INV-4b][运行时主题推送]

```markdown
MainWindow 必须持有已打开模态/非模态弹窗的 `QPointer<T>`。`themeChanged` 信号触发
时，遍历所有 alive pointer 调用 `setTheme(const Theme&)`。该方法实现末尾必须
`ensurePolished() + update()` 以立即生效。
```
