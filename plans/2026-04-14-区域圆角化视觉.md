# 编辑区 / 预览区 / TOC 区域圆角化

## 背景

mockup `tmp/theme_mocks/mock_*.html` 采用玻璃卡片式设计，三个主要区域（编辑、预览、TOC）都带 18-20px 圆角 + 间距，视觉更现代、分区更清晰。当前 Qt widget 全是直角贴边，设计感弱。

## 动作

- [ ] 在 AppWindow 的 central widget 外层加 16px padding，让三个卡片之间露出背景色
- [ ] 为 Editor / Preview / TocPanel 的容器（`QFrame` 或自绘 wrapper）开启圆角：
    - 优先 QSS `border-radius: 18px` + `background` + `border`
    - 内部 QScrollArea 的 viewport 设 `styleSheet("background: transparent")`，`QScrollBar` 单独定制圆角槽
    - 无法用 QSS 搞定的（如 `QTextEdit` 内部滚动条覆盖）用自绘 wrapper：重写 `paintEvent` 按圆角裁剪 + 绘制边框
- [ ] 卡片阴影：可选 `QGraphicsDropShadowEffect`（注意与跨平台性能权衡，低配机提供开关关闭）
- [ ] 菜单栏/标题栏与卡片区之间留 padding，让卡片顶部圆角可见
- [ ] 圆角半径 / 间距 / 阴影强度全部走 Theme 字段，浅色/深色分别调优
- [ ] Spec 更新：`specs/横切关注点/30-主题系统.md` 新增字段 `cardRadius / cardPadding / cardShadow`，列出 INV
- [ ] 与"Tab 右键菜单"等现有 QMenu 兼容测试（菜单弹出定位不被圆角裁剪影响）

## 验收

- 三个区域均显示 18px 圆角，四周留出 16px 间距
- 拖动分隔条调整布局，圆角与阴影不撕裂
- 滚动条贴边时不突破圆角（被裁剪或自身也是圆角）
- 切换浅色↔深色主题，卡片圆角、阴影颜色同步
- 关闭阴影开关后在低配机上 CPU/GPU 占用明显下降

## 风险

- `QGraphicsEffect` 在部分 Linux + X11 + 低版本 Mesa 下有性能坑，必须可关闭
- 自绘 wrapper 切勿覆盖 `WA_TransparentForMouseEvents`，否则子区域丢失点击事件
- 高 DPI 下 border-radius 必须用 px 而不是硬编码；按 `devicePixelRatio` 检查缩放正确（参考 `specs/横切关注点/40-高DPI适配.md`）
- 与 INV-DPI-NO-HARDCODE 协调：圆角 / padding 建议写成 `fm.height() * ratio` 或从 Theme 派生
