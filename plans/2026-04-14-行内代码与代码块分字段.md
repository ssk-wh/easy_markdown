# 预览区：行内代码（`xxx`）与代码块（```xxx```）拆成独立主题字段

## 背景

当前 `Theme::previewCodeBg / previewCodeFg / previewCodeBorder` 三个字段**同时**
用于「行内代码」和「代码块」的渲染：

- `src/preview/PreviewLayout.cpp:375` 行内代码 run 的 bgColor = previewCodeBg
- `src/preview/PreviewPainter.cpp:135` 代码块 fillRect 也用 previewCodeBg

这样主题设计师无法：
- 给行内代码小一号的高亮（例如淡灰底 + 紫字），同时给代码块更强的区隔（例如
  带边框 + 阴影的深底）
- 让行内代码用"连续文字流"的色温，代码块用"独立容器"的色温

## 动作

- [ ] `Theme.h` 新增：
  - `previewInlineCodeBg` / `previewInlineCodeFg`
  - `previewCodeBlockBg` / `previewCodeBlockFg` / `previewCodeBlockBorder`
  - 保留 `previewCodeBg` 等老字段**作为 fallback**，如果新字段未赋值（等于 Theme.h 默认）
    就用老字段。向后兼容老主题 TOML
- [ ] `ThemeLoader.cpp` 的 `kColorTargets` 加：
  - `preview.inline_code_background` / `preview.inline_code_foreground`
  - `preview.code_block_background` / `preview.code_block_foreground` /
    `preview.code_block_border`
- [ ] `PreviewLayout.cpp:375` 行内代码改用 `previewInlineCodeBg`（fallback 到 previewCodeBg）
- [ ] `PreviewPainter.cpp:135` 代码块改用 `previewCodeBlockBg`（fallback 到 previewCodeBg）
- [ ] 8 个内置 TOML 主题按需补新字段（不补就沿用共用值）
- [ ] 更新 `specs/模块-app/12-主题插件系统.md` INV-9 字段清单
- [ ] 单元测试：ThemeLoader 加载新字段 + fallback 链路

## 验收

- 不写新字段的老主题 TOML 依然能加载，视觉与之前一致
- 新字段覆盖时，行内代码与代码块视觉可独立
- 至少 1 个内置主题（如冰霜蓝白）演示两字段独立配色

## 依赖 / 顺序

- 本 plan 属于 Theme 字段扩展，建议与 `plans/2026-04-13-主题插件系统` V2 批次一起做
  （V2 还有：extends 继承、热重载、字段扩展到 80+）

## 风险

- Theme 字段越多 → 用户主题 TOML 的"默认即视觉一致"要求越苛刻，要补默认值
- PreviewLayout / PreviewPainter 两处要同步改，不要漏一处
