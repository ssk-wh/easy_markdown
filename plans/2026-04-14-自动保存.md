# 自动保存

## 背景

写 Markdown 时每次都要 Ctrl+S 才能落盘，心流被打断。主流编辑器（VSCode、Typora、Obsidian）都支持编辑停顿一段时间后静默写盘。本项目当前只有手动保存。

## 动作

- [ ] 设置项新增 `autoSave: bool`（默认 on）与 `autoSaveDelayMs: int`（默认 1500，可选 1500 / 3000 / 5000）
- [ ] Document 层暴露"最后一次编辑时间"信号；AppWindow 持有一个 `QTimer`（singleShot），每次编辑重置计时
- [ ] 计时到期后：**仅对有磁盘路径的 Tab** 执行静默保存（未命名 Tab 不弹文件对话框，跳过）
- [ ] 保存失败（磁盘只读 / 权限 / 外部占用）：状态栏右下显示红色提示 "自动保存失败"，写入日志，不弹对话框打断输入
- [ ] 设置面板加开关与延迟下拉，关闭时完全禁用 QTimer
- [ ] Spec 更新：`specs/模块-app/` 新增"自动保存"章节 + INV（未命名不自动保存 / 失败不打断）+ T 条目

## 验收

- 编辑停顿 1.5s 后已有文件自动落盘，文件 mtime 变化
- 未命名 Tab 停顿 N 秒后**不弹对话框、不落盘**
- 设置关闭 autoSave 后，编辑后不再触发保存
- 将文件改为只读，自动保存失败时状态栏有提示且编辑不阻塞

## 风险

- 与"保存时间戳"显示（见状态栏 plan）联动：自动保存后需刷新状态栏"已保存 · X 分钟前"
- 与外部编辑器同时编辑同一文件时可能冲突，后续可扩展"外部变更检测"
