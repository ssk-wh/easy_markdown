# 应用标题栏居中显示文件名（参考 macOS）

## 背景

macOS 风格和 iOS 26 Liquid Glass mockup（`tmp/theme_mocks/mock_*.html`）的窗口标题都是
**居中显示**（红黄绿三圆点在左，标题居中，右侧空）。
当前 Qt 窗口在 Windows 下标题居左显示（"fm_test.md - SimpleMarkdown"）。

## 动作

- [ ] 调研：Windows 11 原生标题栏是否允许自定义标题对齐（DWM API 层面）
- [ ] 若原生标题栏不支持：实现 custom title bar：
  - `setWindowFlags(Qt::FramelessWindowHint)` + 自绘标题
  - 拖拽/最大化/最小化/关闭三组按钮自己画
  - Windows 下额外处理 WM_NCHITTEST 保留系统级调整大小
- [ ] 风险评估：custom title bar 会丢失 Windows 原生右键菜单（Restore/Minimize/...），需要自行添加
- [ ] Spec 更新：新增「自定义标题栏」章节到 `04-窗口焦点管理.md`
- [ ] 测试矩阵：Windows 10 / 11 / Linux 各验证一次

## 验收

- Windows 下标题栏文件名居中
- 双击标题区仍能最大化/恢复
- 拖拽标题区仍能移动窗口
- 视觉上对齐 Liquid Glass mockup

## 风险

- custom title bar 是常见大坑（滚动条/dock/snap/右键菜单/高DPI 逐个踩）
- 可能需要引入 `QtWinExtras` 或自研 Windows hit-test 逻辑
- 估计工作量 2-3 天
