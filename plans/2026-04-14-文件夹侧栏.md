# 打开目录 + 左侧文件夹侧栏（Markdown 文件树）

## 背景

当前只能一个文件一个文件打开。类似 VSCode 的"打开文件夹"，用户想在 SimpleMarkdown 里：
1. 打开一个目录
2. 左侧出现文件树，列出该目录下所有 `.md` / `.markdown` 文件（递归或单层可选）
3. 单击文件名 → 用当前活动 Tab 打开；双击 → 新建 Tab 打开
4. 右键文件：新建、重命名、删除、在系统文件管理器中显示

这能让 SimpleMarkdown 从"打开文件"的工具上升为"管理 Markdown 工作区"的工具。

## 动作

- [ ] 菜单：File → Open Folder...（`Ctrl+K Ctrl+O` 类似 VSCode）
- [ ] 主窗口加一个左侧 Dock 或 Splitter 面板：`FolderPanel`
  - `QTreeView` + `QFileSystemModel`（filter `*.md *.markdown`）
  - 根路径显示在顶部标题条（见 `plans/2026-04-14-编辑预览目录区域标题条.md`）
- [ ] 单击/双击行为区分
- [ ] 文件监控：外部新增/删除/重命名文件时 tree 自动刷新（QFileSystemWatcher 或 Model 内置）
- [ ] 持久化："上次打开的文件夹"，启动时恢复（会话恢复一起做）
- [ ] 右键菜单：New File / New Folder / Rename / Delete / Reveal in Explorer / Copy Path
- [ ] Spec：新建 `specs/模块-app/13-文件夹侧栏.md`

## 验收

- 能打开一个 5MB / 500 个 md 文件的目录，响应流畅
- 单击文件名能正确切换到当前活动 Tab，双击能开新 Tab
- 外部删除/重命名文件，tree 3 秒内同步
- 右键菜单所有操作可用且不崩溃
- 主题切换时 tree 样式跟随（不硬编码）

## 风险

- `QFileSystemModel` 在 Windows 大目录下扫描会卡 UI → 需要异步 populate
- 文件重命名时编辑器里打开的对应 Tab 需要更新路径
- 与现有 "Recent Files" / "Open File..." 的交互需要明确（并存，不冲突）
