## Feature

打开渲染文件时，只需要渲染可见区域内容以及前后约 10 行内容，其他区域不渲染，从而加快渲染速度提升性能。

## 现状分析

当前 paintBlock 已有视口裁剪（culling）：
```cpp
if (blockBottom < scrollY - 50 || absY > scrollY + viewportHeight + 50) {
    countBlockChars(block);
    return;
}
```

即超出视口±50px 的 block 已跳过绘制，仅做 charCounter 计数。

## 可进一步优化的方向

1. **布局阶段优化**：`buildFromAst` 目前计算所有 block 的布局（bounds），对大文件耗时。可改为增量/虚拟布局——只精确计算可见区域附近 block 的位置，远处 block 用估算高度占位。
2. **扩大 culling 边距**：当前 ±50px 可改为 ±N 行高度，确保快速滚动时无闪烁。
3. **异步预计算**：在后台线程预先计算远处 block 的精确高度，滚动接近时替换估算值。
