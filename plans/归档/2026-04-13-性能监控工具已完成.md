# 内置性能监控工具

## 背景
Spec 已定义性能预算（如布局/绘制延迟、滚动 FPS），但缺乏运行时手段验证、定位瓶颈。开发者需要内置工具看到实际的耗时分布。

## 动作
- [ ] 引入轻量计时工具（`QElapsedTimer` 包装宏）
- [ ] 在关键路径埋点：解析、布局、绘制、滚动、搜索
- [ ] 调试模式下提供"性能"面板，显示最近 N 次操作的耗时分布
- [ ] 超过 Spec 预算时在面板中红色标注
- [ ] 默认 Release 构建关闭面板，仅保留极轻的计数

## 验收
- Debug 构建打开面板能实时看到各阶段耗时
- 故意触发卡顿（大文档 + 频繁滚动）能看到对应阶段超预算告警
- Release 构建无可观察的额外开销
