# Theme 菜单项未国际化

## 背景

`src/app/MainWindow.cpp` 的 `rebuildThemeMenu` 里，主题子菜单的多项菜单项
使用 `tr()` 但**实际不会触发翻译**，因为：

1. 菜单项文字 `tr("Theme")` / `tr("Follow System")` / `tr("Open Themes Folder")` /
   `tr("Rescan Themes")` — 这几条如果 `lupdate` 扫不到或 `.ts` 里没条目就不翻
2. **主题 display name**（从 TOML `meta.name` 读）直接用 `addAction(d.displayName)`，
   **没有走 `tr()`**。用户若把 locale 切成英文，中文主题名仍然显示中文。反之亦然。

对应 memory 里已有的警示：
> 「新增 UI 必须走完 i18n 全链路」: tr() + lupdate + 补翻译 + lrelease + 打包含 .qm

## 动作

- [ ] 确认所有 Theme 子菜单项都用 `tr()`：Theme / Follow System / Open Themes Folder
      / Rescan Themes
- [ ] **主题 display name 多语言化**：参考 `plans/归档/2026-04-14-主题中文名.md` 的 V2 方案——
      TOML 里写 `name` + `name_en_US` + `name_zh_CN`，ThemeLoader 按 QLocale 选一个
      （或者更灵活：`[meta.name] default = "..."` + `[meta.name] en_US = "..."` 子表）
- [ ] 运行 `lupdate` 刷新 `translations/simple_markdown_zh_CN.ts` / `_en_US.ts`
- [ ] 补中英文翻译条目
- [ ] 运行 `lrelease` 生成 `.qm` 并确认资源打包（`resources.qrc`）
- [ ] Spec 更新：`specs/横切关注点/60-国际化.md` 增补主题菜单 i18n 要求
- [ ] 测试矩阵：中文 locale / 英文 locale 各看一次 Theme 子菜单

## 验收

- 中文 UI 下菜单项为「主题 / 跟随系统 / 打开主题目录 / 重新扫描主题」
- 英文 UI 下菜单项为「Theme / Follow System / Open Themes Folder / Rescan Themes」
- 主题 display name 按 locale 切换（"冰霜蓝白" ↔ "Arctic Frost"）
- 菜单切换主题功能不受 locale 影响

## 风险

- 主题 display name 的多语言方案会影响 TOML schema，需要版本兼容考虑
  （老用户 TOML 只有 `name` 字段时仍能加载）
- lupdate 对代码改动敏感，打包流程若漏掉 `.qm` 会导致用户看不到翻译
