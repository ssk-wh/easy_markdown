# 预览区代码块字号明显比编辑器小

## 背景

用户反馈：同一文档里，编辑器显示的 ```cpp ``` 代码块内容字号和预览区渲染出来的代码块
字号**明显不一致**——预览的代码字更小，读起来吃力、与编辑器视觉重心不协调。

## 根因假设（实施时需定位确认）

`src/preview/PreviewPainter.cpp` / `PreviewLayout.cpp` 里代码块绘制可能：
- 使用了独立的 `codeFont` 常量，字号未跟随编辑器 `EditorLayout::m_font.pointSize()`
- 或使用了 `font.setPointSize(font.pointSize() - N)` 人为缩小
- 或受 `FontDefaults.h` 字号派生链影响，代码比正文小（但偏差过大）

Spec: `specs/横切关注点/80-字体系统.md` 已经要求「字号统一从中心化常量派生」。
需要检查代码块渲染是否绕过了这个中心化来源。

## 动作

- [ ] 定位代码块字号设置点：
  - `grep -rn "codeFont\|QFont.*[Mm]ono" src/preview/`
  - 查 `FontDefaults.h` 的 `kPreviewCodeFontSize` / `kEditorCodeFontSize` 等常量
- [ ] 对齐规则：预览代码块字号 **= 预览正文字号 × 1.0**（保持视觉重心与编辑器一致），
      或更精确：**与编辑器代码字号一致**（因用户在编辑器写代码，预览应等价呈现）
- [ ] 测试三档 DPI（1x / 1.25x / 1.5x）下字号比例一致
- [ ] 测试放大/缩小（Ctrl+=/Ctrl+-）后编辑器与预览代码块字号同步变化
- [ ] Spec 增补 `specs/横切关注点/80-字体系统.md`：
      新增 INV「预览代码块字号 = 编辑器代码字号」+ 对应 T 编号
- [ ] 回归测试：FrontmatterRenderTest 或新增 CodeBlockFontSizeTest 验证

## 验收

- 同一文档里编辑器和预览里 ```cpp 代码块 ``` 内容字符**高度目测一致**（±1px 像素级）
- Ctrl+= 放大后两侧同步
- 切换不同主题（字号不应受主题影响）字号也保持一致

## 风险

- 字号改动会轻微影响布局（代码块行高），FrontmatterRenderTest 相关坐标断言可能需要更新
- 高 DPI 下需要注意 INV-DPI-METRICS（QFontMetricsF 必须传 QPaintDevice*）
