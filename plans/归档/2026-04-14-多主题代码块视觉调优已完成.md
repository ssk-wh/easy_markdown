# 多主题代码块配色调优（去除"反色"反模式）

## 背景

用户反馈 3 个主题的代码块背景深色太重，与整体浅色主题基调不搭：

1. **冰霜蓝白（Arctic Frost）**：预览区 ```代码块``` 背景是深墨 `#0E1925`，
   字体 `#CCD4E4` 淡灰，看不清——与浅色整体审美冲突
2. **琥珀桃暖（Sunset Haze）**：预览区代码块背景 `#2E1913` 深栗墨，字 `#F0DBC1`
   暖米，同样与暖白主题冲突
3. **墨禅单色（Monochrome Zen）**：**编辑区**代码块背景变纯黑 `#1A1A1A`，
   字 `#F0EDE6` 浅奶，与宣纸墨书的整体浅色调完全冲突

这是 TOML 设计时刻意做的"反色对比"审美选择，但实测造成：
- 视觉重量失衡，浅色主题整屏多一块过重的深色方块
- 代码字小 + 反色阅读负担更重
- 与各主题的"整体色温/明度"破坏掉

## 动作（纯配置改动，无架构改动）

- [ ] `resources/themes/arctic-frost.toml`：
  - `preview.code_background` `#0E1925` → `#ECF3FB`（浅冰蓝，与 mock_a 一致）
  - `preview.code_foreground` `#CCD4E4` → `#1F3A5F`（深海蓝，可读）
  - `preview.code_border` `#1A2A40` → `#CFE0F0`
- [ ] `resources/themes/sunset-haze.toml`：
  - `preview.code_background` `#2E1913` → `#FBEADA`（浅暖米）
  - `preview.code_foreground` `#F0DBC1` → `#3A1F14`（深栗）
  - `preview.code_border` `#4A2A1E` → `#F3D9BD`
- [ ] `resources/themes/monochrome-zen.toml`（编辑器 + 预览都去反色）：
  - `syntax.code_block` `#F0EDE6` → `#1A1A1A`（字变回深墨）
  - `syntax.code_block_background` `#1A1A1A` → `#EEE9DF`（底变回浅奶）
  - `preview.code_background` `#1A1A1A` → `#EEE9DF`
  - `preview.code_foreground` `#F0EDE6` → `#1A1A1A`
  - `preview.code_border` `#3A3530` → `#D6D2C8`
- [ ] 编译 + 切主题截图对比，验证 3 个主题代码块视觉与主题色温一致
- [ ] 深夜极光（Midnight Aurora）保持不变（本身深色主题）

## 验收

- 3 个浅色主题下 ``` 代码块 ``` 背景是**浅色**，字是**深色**
- `行内代码` 和 ``` 代码块 ``` 的色温一致（都浅底）
- 深夜极光下代码块保持深紫（`#0D0820`）
- 单元测试 `ThemeLoaderTest` 继续绿

## 风险

- 改动后视觉与 UI Mockup HTML 原稿会稍有出入（mock_a/b/f 里代码块是浅色的，
  所以实际是向 mock 对齐，反而更一致）
