# 深色主题下菜单栏和 TOC 面板不跟随主题（保持默认灰黑）

## 背景

用户设置"深夜极光"（Midnight Aurora）后发现：
- **菜单栏**完全是默认深色灰（`#2b2b2b` 左右），**不是**主题定义的深紫调（`#140E28` / `#1A1233`）
- **右侧目录（TOC）**也是默认深色，与主题的深紫极光基调不搭

## 根因

1. **`src/app/MainWindow.cpp` 的 `applyTheme` 里 `if (theme.isDark)` 分支**是硬编码
   stylesheet（保留了 Spec INV-4 弹窗覆盖要求）：
   ```cpp
   "QMainWindow { background: #2b2b2b; }"
   "QMenuBar { background: #2b2b2b; color: #ccc; border: none; }"
   "QTabBar { background: #2b2b2b; border: none; }"
   ...
   ```
   所有深色主题（内置 Dark、Midnight Aurora、以及未来任何 `is_dark = true` 的用户主题）
   都走同一段灰黑 `#2b2b2b`，**不会**根据 Theme 字段变色
2. **`src/preview/TocPanel.cpp`** 的 `setTheme` 实现可能只用了 `previewBg/Fg`，或直接
   使用默认 palette，未完整数据化（需进一步审查）

## 动作

- [ ] **Dark 分支 stylesheet 数据化**（对称非 dark 分支的既有改造）：
  - 把 `#2b2b2b` / `#3c3f41` / `#1e1e1e` / `#4a9eff` 等硬编码色替换成从 Theme 派生：
    - `QMainWindow background` → `theme.editorBg`
    - `QMenuBar background` → `theme.editorGutterBg`
    - `QMenuBar::item:selected background` → `theme.editorCurrentLine`
    - `QMenuBar::item:selected border-bottom` → `theme.accentColor`
    - `QMenu background / color / border` → 同上衍生
    - `QTabBar` 系列 → 同 menuBar
    - `QScrollBar` → `theme.editorGutterLine` / `theme.editorLineNumber` 衍生
  - **保留** `QDialog / QMessageBox / QTextEdit / QPushButton` 等弹窗相关 stylesheet 硬编码
    （Spec INV-4 强要求：深色模式下弹窗全部跟随，避免 Qt 5.15 QDialog 默认浅色 bug）
  - 更谨慎一点：保留深色弹窗硬编码但主题可以 override（比如 isDark 下走旧硬编码 +
    新叠加 theme 字段染色）
- [ ] **TocPanel 完全数据化**：
  - 审查 `src/preview/TocPanel.cpp` 的 `setTheme` 和 paintEvent
  - 确保背景、当前项高亮、层级缩进线、hover 态全部读 Theme 字段
  - 至少测试：切到"深夜极光"后 TOC 背景应是深紫 `#1A1233`，不是灰黑
- [ ] Spec 更新：`specs/横切关注点/30-主题系统.md` INV-4 区分"弹窗数据化例外"与"主窗口/TOC
      必须数据化"
- [ ] 验证矩阵：Dark / 深夜极光 各切一次，截图对比 menuBar/TabBar/TOC/StatusBar 全部是
      对应主题色，而非 #2b2b2b 通用灰

## 验收

- 深夜极光下 menuBar 是深紫色（接近 theme.editorGutterBg `#1A1132`），不是灰黑 #2b2b2b
- 深夜极光下 TOC 面板是深紫色，当前项 accent 是霓虹紫 `#A78BFA`
- 内置 Dark 主题视觉保持与旧版本一致（或只有微小差异，因 editorGutterBg 等接近 #1E1E1E）
- 弹窗（关于/快捷键/更新日志/文档统计）在 Dark 下仍全部深色跟随，不回到浅色

## 风险

- Dark 分支 stylesheet 数据化后，`QDialog` 深色弹窗可能漏项（Spec INV-4 陷阱再现）
- TocPanel 若用 palette 绘制，需要同时 setPalette + setStyleSheet（INV-5）
- 深色主题太多时视觉可能差异微妙，需要 UI 审查
