# 状态栏右下去掉当前主题名 label

## 背景

状态栏右下目前显示四个元数据 label：
`UTF-8 | LF | <主题名> | 已保存 · 刚刚`

用户反馈：**主题名不需要**。理由（推测）：
- 用户切换主题后立刻能从整个 UI 颜色感知到当前主题
- 主题名占屏幕宽度，状态栏应保持信息密度高
- 菜单 视图 → 主题 里已有打勾的当前主题，查找成本不高

## 动作

- [ ] `MainWindow.h` 删除成员 `QLabel* m_statusThemeName`
- [ ] `MainWindow.cpp::setupStatusBar` 去掉 `makeRightLabel(QString())` 那行
- [ ] `MainWindow.cpp::updateRightStatusBar` 去掉 `m_statusThemeName->setText(...)` 那行
- [ ] `specs/模块-app/15-状态栏布局.md`：
  - 修改 INV-1（右区 label 从 4 个减为 3 个）
  - 删除 INV-4（主题名字段）相关条目
  - 删除 T-SB-9（切换主题刷新主题名）
- [ ] CHANGELOG 加 Changed

## 验收

- 状态栏右下从左到右显示：`UTF-8 | LF | 已保存 · 刚刚`，不再有主题名
- 切换主题后状态栏内容保持为 3 个 label，不出现空白位
- 回归：自动保存失败提示 label 依然在（它跟在 3 个 label 之后）

- 无外部依赖；单一文件修改
