# 编辑器 / 预览 / TOC 区域上方的标题条 + 操作按钮

## 背景

参考 `tmp/theme_mocks/mock_a_arctic_frost.html` 的效果，每个主区域上方都有一个**标题条**：

- 编辑器区域：左侧显示"📝 编辑器 · EDITOR"，右侧有撤销/重做/保存按钮
- 预览区域：左侧显示"👁 预览 · PREVIEW"，右侧有"实时"状态指示灯
- TOC 目录：左侧显示"📑 目录 · OUTLINE"

当前这些区域没有标题条，直接贴内容。加了标题条后：
- 视觉更有层次（符合 Liquid Glass 分层设计）
- 操作按钮（保存/撤销/重做）有明确归属位置，不挤在菜单栏里
- 用户一眼知道每块是什么

## 动作

- [ ] 新建 `src/app/PanelHeader.h/.cpp`：通用标题条组件（图标 + 文字 + 右侧按钮槽）
- [ ] 编辑器、预览、TOC 外包一层 `QVBoxLayout`，在内容顶部插入 `PanelHeader`
- [ ] 编辑器标题条右侧按钮：撤销、重做、保存（快捷键与 File/Edit 菜单一致）
- [ ] 预览标题条右侧：一个圆点 + "实时" 文字（绿色=同步中，黄色=防抖延迟中，红色=解析失败）
- [ ] TOC 标题条右侧：一个"折叠全部"按钮（需要先做完 TOC 折叠）
- [ ] 标题条配色全部走 `Theme` 字段（不硬编码颜色）
- [ ] Spec：分别在三个模块的 README 里加 INV + T 条目

## 验收

- 三个区域都有视觉一致的标题条
- 编辑器的保存/撤销/重做按钮与菜单项行为等价
- 主题切换时标题条也跟随变色（不硬编码）
- 标题条高度 ≤ 32px，不吃内容区空间
